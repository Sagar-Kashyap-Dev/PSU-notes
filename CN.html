1. Network Baselines & Reference Models

LAN: High-Speed Local Domain
WAN: Broad Inter-City Spans
OSI: 7-Layer Conceptual Stack
TCP/IP: 4/5-Layer Implementable Suite
Topological Scale Categories
  • LAN (Local Area Network): Spans small areas like homes or offices. Offers high data transfer rates, minimal propagation delays, and low error rates, typically utilizing Ethernet or Wi-Fi.
  • MAN (Metropolitan Area Network): Extends across a town or entire city (e.g., cable television networks).
  • WAN (Wide Area Network): Covers long distances across countries or continents. Suffers from higher propagation delays and relies on leased lines, fiber backbones, and satellite links.

2. Data Link Layer (Framing & Error Engineering)

Framing: Bit & Byte Stuffing Boundary Checks
CRC: Polynomial Modulo-2 Engineering
Hamming: Single Error Correcting Matrix
Hamming Dist: $d+1$ Detect / $2d+1$ Correct
Framing Strategies
  • Character Count: Uses a header field to specify the frame length. Highly vulnerable to corruption; if a count bit changes, the receiver loses alignment with subsequent frames.
  • Byte Stuffing: Delineates frames using dedicated flag bytes. If a flag pattern occurs naturally within the payload, a special escape byte (`ESC`) is stuffed before it.
  • Bit Stuffing: Uses a unique bit pattern flag (typically `01111110`). Whenever the transmitter encounters five consecutive `1`s in the payload, it automatically injects a dummy `0` to prevent accidental flag creation.
Error Detection via Cyclic Redundancy Check (CRC)

Given a data bitstream string $D$ and a generator polynomial divisor $G$ of degree $r$:

  1. Append $r$ zeros to the end of the data payload string $D$.
  2. Perform **Modulo-2 Binary Division** (using bitwise XOR operations instead of standard subtraction) on the appended data string using the divisor $G$.
  3. The resulting remainder of length $r$ bits replaces the appended zeros to form the transmitted frame. If the receiver obtains a non-zero remainder upon division, an error has occurred.
Hamming Code Error Correction Geometry

Enables Single-Error Correction (SEC) by inserting parity bits at power-of-2 index locations ($1, 2, 4, 8, \dots$).

  • Parity Structural Rule Inequality: For $m$ data bits and $p$ parity bits, the relation must satisfy: $$2^p \ge m + p + 1$$
  • Distance Mechanics: To detect $k$ errors, the minimum Hamming distance between any two valid code words must be at least **$k + 1$**. To correct $t$ errors, the minimum distance must be at least **$2t + 1$**.

3. Medium Access Control (MAC) Protocols

Pure ALOHA: 18.4% Max Efficiency Bound
Slotted ALOHA: 36.8% Max Efficiency Bound
CSMA/CD: Carrier Sense Collision Detect
Min Frame Rule: $L \ge 2 \cdot R \cdot T_p$
ALOHA Variations
  • Pure ALOHA: Stations transmit frames immediately whenever data is ready. Collisions occur if any part of a frame overlaps with another. Maximum throughput is bounded at: $$\eta = G \cdot e^{-2G} \implies \eta_{\text{max}} = \frac{1}{2e} \approx 18.4\%$$
  • Slotted ALOHA: Divides time into discrete slots matching a standard frame transmission time. Stations can only begin transmitting at the start of a time slot, cutting the vulnerable period in half. Maximum throughput: $$\eta = G \cdot e^{-G} \implies \eta_{\text{max}} = \frac{1}{e} \approx 36.8\%$$
CSMA/CD Engine Mechanics

Stations listen to the channel before transmitting ("carrier sense"). If the channel is busy, they defer; if idle, they transmit while continuing to listen to detect collisions.

Critical Frame Length Constraint Equation:
  • To ensure a transmitting station can reliably detect a collision before it finishes sending its frame, the frame length must satisfy the minimum size requirement: $$\text{Frame Length } (L) \ge 2 \cdot \text{Bandwidth } (R) \cdot \text{Propagation Delay } (T_p)$$

4. Network Layer (IPv4, IPv6 & Subnetting)

IPv4 Address Space: 32-Bit Linear Space
IPv6 Address Space: 128-Bit Hex Space
CIDR Notation: Variable Length Subnet Mask
Usable Host Formula: $2^H - 2$ Addresses
IP Addressing Protocols
  • IPv4 Header: 32-bit source and destination addresses. Key fields include: *Time to Live (TTL)* to prevent routing loops, *Header Checksum* for integrity verification, and fields for *Fragmentation* (Identification, Flags, Fragment Offset).
  • IPv6 Header: 128-bit source and destination addresses. Simplifies routing by using a fixed 40-byte base header and removing the header checksum field.
Subnet Arithmetic Engine

Classless Inter-Domain Routing (CIDR) groups IP addresses into blocks using variable-length masks (e.g., `/24`).
For a network mask with $H$ bits allocated for the host portion:

  • Total address space allocations within the block = $2^H$
  • Total usable host addresses per subnet: $$2^H - 2$$ *(Subtracts the Network Address where all host bits are 0, and the Directed Broadcast Address where all host bits are 1).*

6. Transport Layer (TCP vs. UDP Architecture)

TCP: Connection-Oriented Byte Stream
UDP: Connectionless Unreliable Datagram
3-Way Handshake: SYN → SYN-ACK → ACK Matrix
Pseudo Header: Cross-Layer UDP Integrity Check
Transport Protocols Comparison
  • TCP (Transmission Control Protocol): Reliable, connection-oriented, and byte-stream based. Enforces delivery tracking using sequence and acknowledgment numbers. Features error validation, flow management, and congestion control.
  • UDP (User Datagram Protocol): Lightweight, connectionless, and unreliable. Minimal 8-byte overhead header (Source Port, Destination Port, Length, Checksum). Preferable for real-time video and DNS lookups where speed is prioritized over retransmission delays.
TCP Connection Control States
  • Establishment (Three-Way Handshake): `SYN` $\to$ `SYN-ACK` $\to$ `ACK`. Coordinates initial sequence numbers (ISNs) securely between hosts.
  • Teardown (Four-Wave Closure): `FIN` $\to$ `ACK` $\to$ `FIN` $\to$ `ACK`. Requires a `TIME_WAIT` state to ensure any delayed packets remaining in the network clear before the socket closes.

7. Flow Control, Windows & Congestion Management

Stop & Wait Efficiency: $\eta = 1 / (1 + 2a)$
Sliding Window Size: $1 + 2a$ Upper Bound
Slow Start: Exponential CWND Increase
AIMD Phase: Additive Up / Multiplicative Down
Flow Control Efficiency Mathematics

Let $a = \frac{T_p}{T_t}$, where $T_p$ is propagation delay and $T_t$ is packet transmission delay ($T_t = \frac{\text{Packet Size}}{\text{Bandwidth}}$):

  • Stop-and-Wait Efficiency: $$\eta = \frac{1}{1 + 2a}$$
  • Sliding Window Utilization: To achieve $100\%$ link utilization ($\eta = 1$), the sender's window size ($W$) must satisfy: $$W \ge 1 + 2a$$
  • Go-Back-N (GBN) vs. Selective Repeat (SR) Window Constraints:
    • *GBN:* Sender Window = $2^n - 1$, Receiver Window = $1$ (Discards out-of-order packets).
    • *SR:* Sender Window = $2^{n-1}$, Receiver Window = $2^{n-1}$ (Buffers out-of-order packets).
TCP Congestion Control States Engine

The Congestion Window (`cwnd`) dynamically adapts based on network indicators:

  1. Slow Start Phase: `cwnd` starts at 1 MSS and **doubles every Round Trip Time (RTT)** until it reaches the slow-start threshold (`ssthresh`).
  2. Congestion Avoidance (AIMD): Once `cwnd` $\ge$ `ssthresh`, it increases **linearly by 1 MSS per RTT**.
  3. Timeout Recovery: If a timeout occurs, `ssthresh` is set to $\frac{\text{cwnd}}{2}$, and `cwnd` resets to 1 MSS, restarting the Slow Start phase.
  4. Fast Recovery (3 Duplicate ACKs): Triggered by receiving three duplicate ACKs. The system drops `ssthresh` to $\frac{\text{cwnd}}{2}$ and sets `cwnd` equal to `ssthresh`, entering Congestion Avoidance directly instead of resetting to 1.

8. Application Infrastructure & Network Hardware

DNS: UDP 53 Name Translation
DHCP: UDP 67/68 IP Allocation
Switch: Layer 2 ASIC Collision Separator
Router: Layer 3 Domain Isolation Engine
Application Layer Protocols Core Profiles
  • DNS (Domain Name System): Translates domain names to IP addresses. Operates primarily over UDP port 53.
  • DHCP (Dynamic Host Configuration Protocol): Dynamically assigns IP addresses, subnet masks, and default gateways to joining hosts. Relies on a four-step lease process: Discover, Offer, Request, Acknowledgment (DORA).
  • Mail Delivery Suite: **SMTP** pushes emails from clients to servers or between mail servers (TCP port 25). **POP3** (TCP port 110) and **IMAP** (TCP port 143) pull emails from servers to local clients.
  • Web Access Layer: **HTTP** (TCP port 80) transmits plaintext web pages. **HTTPS** (TCP port 443) secures data transfers by encrypting traffic using Transport Layer Security (TLS).
