1. Complexity Analysis & Asymptotic Bounds

Big O: Asymptotic Upper Bound
Omega (Ω): Asymptotic Lower Bound
Theta (Θ): Strict Tight Bound
Little o: Non-Tight Upper Bound
Formal Asymptotic Definitions
  • Big O ($O$): $f(n) = O(g(n))$ if there exist positive constants $c$ and $n_0$ such that $0 \le f(n) \le c \cdot g(n)$ for all $n \ge n_0$. Represents the structural worst-case envelope.
  • Omega ($\Omega$): $f(n) = \Omega(g(n))$ if there exist positive constants $c$ and $n_0$ such that $0 \le c \cdot g(n) \le f(n)$ for all $n \ge n_0$. Defines the asymptotic minimum threshold.
  • Theta ($\Theta$): $f(n) = \Theta(g(n))$ if there exist positive constants $c_1$, $c_2$, and $n_0$ such that $0 \le c_1 \cdot g(n) \le f(n) \le c_2 \cdot g(n)$ for all $n \ge n_0$. Holds if and only if $f(n) = O(g(n))$ and $f(n) = \Omega(g(n))$.
  • Little o ($o$): $f(n) = o(g(n))$ if for **any** positive constant $c > 0$, there exists an $n_0 > 0$ such that $0 \le f(n) < c \cdot g(n)$ for all $n \ge n_0$. Alternatively: $$\lim_{n \to \infty} \frac{f(n)}{g(n)} = 0$$
Asymptotic Dominance Growth Hierarchy

Arranged from lowest to highest growth rate (most to least efficient runtime profile):

$$\mathcal{O}(1) < \mathcal{O}(\log \log n) < \mathcal{O}(\log n) < \mathcal{O}(\sqrt{n}) < \mathcal{O}(n) < \mathcal{O}(n \log n) < \mathcal{O}(n^2) < \mathcal{O}(n^3) < \mathcal{O}(2^n) < \mathcal{O}(n!)$$
Critical Analytical Traps:
  • Conditional Bound Violations: Asymptotic comparisons are only valid as $n \to \infty$. Do not make performance assumptions based on small values of $n$. For instance, $1000n$ grows significantly faster than $0.01n^2$ for small values, but asymptotically $0.01n^2$ dominates.
  • Little o vs. Big O Distinction: Big O includes the possibility of the functions growing at the exact same rate ($f(n) = \Theta(g(n))$), whereas Little o strictly excludes tight matches, requiring $f(n)$ to grow strictly slower than $g(n)$.

2. Recurrence Relations

Substitution: Inductive Verification
Iteration: Expansion Tree Summation
Master Theorem: Divide & Conquer Solver
Extended Case: Logarithmic Density Handling
Resolution Methodologies
  • Substitution Method: Formulate a guess for the runtime bound, then use mathematical induction to prove the guess holds across the boundary conditions.
  • Iteration Method (Recursion Tree): Expand the recurrence relation into a tree structure. Calculate the total work done at an arbitrary level $i$, find the maximum depth of the tree, and sum the work across all levels to find the overall complexity.
The Standard Master Theorem Blueprint

Applies directly to divide-and-conquer recurrences of the form: $T(n) = aT(n/b) + f(n)$, where $a \ge 1$ and $b > 1$.

Extended Master Theorem (Logarithmic Cases)

When $f(n) = \Theta(n^{\log_b a} \log^k n)$ for $k \ge 0$:

$$T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$$
Critical Analytical Traps:
  • Master Theorem Non-Applicability: The Master Theorem cannot be used if $a$ is not a constant (e.g., $T(n) = n T(n/2)$), if the subproblem reduction is sub-linear (e.g., $T(n) = T(n-1) + 1$), or if the ratio $\frac{f(n)}{n^{\log_b a}}$ is not polynomial (e.g., $T(n) = 2T(n/2) + n \log n$, where Case 3 doesn't apply because the difference isn't polynomial; this requires the extended logarithmic case instead).

  • Linked List Binary Search Trap: Performing a binary search on a sorted **Singly Linked List** does not run in $\mathcal{O}(\log n)$ time. Fast binary search requires $O(1)$ random access to elements, such as in an array.

Core Property Definitions
  • Stable: Preserves the relative input order of elements with equal keys.
  • In-place: Does not require additional memory that scales with the size of the input; auxiliary space is bounded at $\mathcal{O}(1)$.
  • Adaptive: Runs faster when given input that is already partially or fully sorted.
Critical Analytical Traps:
  • Quick Sort Degeneration: Quick Sort drops to its worst-case performance of $\mathcal{O}(n^2)$ if the input array is already sorted or reverse-sorted and the algorithm always picks the first or last element as the pivot. This causes highly unbalanced partitions of size $0$ and $n-1$.
  • Comparison Sort Theoretical Boundary: Any sorting algorithm that relies solely on comparing elements must take at least $\Omega(n \log n)$ time in the worst case. Non-comparison algorithms like Radix Sort bypass this limit because they use element properties (like digit distributions) instead of comparisons.

5. Divide & Conquer vs. Greedy Strategy

D&C Strategy: Independent Subproblems
Greedy Choice: Local Optimal Choice
Job Sequencing: Sorted Deadlines Max Profit
Huffman Space: Prefix-Free Variable Encoding
Divide & Conquer Architecture

Breaks a problem down into independent subproblems, recursively solves them, and combines their results. Examples include Merge Sort, Quick Sort, and Binary Search.

Greedy Paradigm & Core Applications

Makes the locally optimal choice at each step, hoping it leads to a globally optimal solution. It relies on the **Greedy Choice Property** and **Optimal Substructure**.

  • Job Sequencing with Deadlines: Sort all available jobs in descending order of profit. Attempt to schedule each job into its latest possible free time slot before its deadline. Runs in $\mathcal{O}(n^2)$ or $\mathcal{O}(n \log n)$ with disjoint-set optimizations.
  • Huffman Coding Framework: A variable-length compression technique. Iteratively merges the two leaf nodes with the lowest frequencies into a parent node whose frequency is the sum of its children's frequencies. Constructs a prefix-free binary tree in $\mathcal{O}(n \log n)$ time.
Critical Analytical Traps:
  • Fractional vs. 0/1 Knapsack: The greedy approach works perfectly for the **Fractional Knapsack** problem (by selecting items based on their profit-to-weight ratio). However, it **fails to guarantee an optimal solution for the 0/1 Knapsack problem**, which requires a Dynamic Programming approach because choices cannot be reversed or split.

6. Dynamic Programming & Backtracking

DP Condition: Overlapping Subproblems
0/1 Knapsack Time: O(n * W) Pseudo-polynomial
LCS Matrix Bound: O(m * n) Time/Space
Backtracking: State-Space Tree Pruning
Dynamic Programming Architecture

Solves problems by combining solutions to overlapping subproblems, storing results in a lookup table (memoization or tabulation) to avoid redundant calculations.

  • 0/1 Knapsack Problem: The state equation evaluates whether to include or exclude an item based on remaining capacity: $$V[i, w] = \max(V[i-1, w], v_i + V[i-1, w - w_i])$$ Runs in $\mathcal{O}(n \cdot W)$ time and space, where $W$ is the maximum weight capacity.
  • Longest Common Subsequence (LCS): Compares string characters to build a matching sequence: $$L[i, j] = \begin{cases} L[i-1, j-1] + 1 & \text{if } X[i] == Y[j] \\ \max(L[i-1, j], L[i, j-1]) & \text{if } X[i] \neq Y[j] \end{cases}$$
  • Matrix Chain Multiplication (MCM): Determines the most efficient way to multiply a chain of matrices by finding the optimal split point $k$: $$M[i, j] = \min_{i \le k < j} \{ M[i, k] + M[k+1, j] + p_{i-1}p_k p_j \}$$ Runs in $\mathcal{O}(n^3)$ time and requires $\mathcal{O}(n^2)$ auxiliary space.
Backtracking Strategy (N-Queen)

Explores a state-space tree systematically using Depth-First Search. If a path violates the problem constraints, the algorithm **prunes the branch** and backtracks to the previous decision point.

  • N-Queens Problem: Places $N$ queens on an $N \times N$ chessboard so that no two queens threaten each other. Checks row, column, and diagonal constraints at each step.
Critical Analytical Traps:
  • Pseudo-Polynomial Time Reality: The $\mathcal{O}(n \cdot W)$ runtime of the 0/1 Knapsack problem is **pseudo-polynomial**, not truly polynomial. If the capacity $W$ scales exponentially with the input size ($W = 2^n$), the runtime becomes exponential ($\mathcal{O}(n \cdot 2^n)$).

7. Graph Systems & Traversals

BFS / DFS: O(V + E) Complexity
Prim's: Vertex Growth MST
Kruskal's: Sorted Edge Disjoint MST
Dijkstra: Single Source Shortest Path
Graph Traversals
  • BFS: Uses a queue to explore neighbors level by level. Ideal for finding the shortest path in unweighted networks. Complexity: $\mathcal{O}(V + E)$.
  • DFS: Uses a stack or recursion to explore as deep as possible along each branch. Used to identify connected components, perform topological sorting, and detect cycles. Complexity: $\mathcal{O}(V + E)$.
Minimum Spanning Tree (MST) Engines
  • Prim's Algorithm: Starts from an arbitrary root vertex and grows the tree by adding the cheapest edge that connects a vertex in the tree to a vertex outside the tree. Runs in $\mathcal{O}(E \log V)$ using a binary heap pointer.
  • Kruskal's Algorithm: Sorts all edges in the graph by weight and greedily adds the cheapest edge, provided it doesn't create a cycle. Uses a **Disjoint-Set Data Structure (Union-Find)** to detect cycles efficiently. Runs in $\mathcal{O}(E \log E)$ or $\mathcal{O}(E \log V)$ time.
Dijkstra's Shortest Path Algorithm

Computes the shortest path from a single source vertex to all other vertices in a weighted graph. Maintained using a min-priority queue. Runs in $\mathcal{O}(E \log V)$ time.

Critical Analytical Traps:
  • Dijkstra's Negative Weight Failure: Dijkstra's algorithm **cannot handle graphs with negative edge weights**. It locks in the shortest distance to a vertex as soon as it is extracted from the priority queue; a subsequent negative edge could offer a shorter path that the algorithm will miss. Graphs with negative weights require the Bellman-Ford algorithm instead.

8. NP-Completeness Matrix

P: Polynomial Time Deterministic
NP: Polynomial Time Verifiable
NP-Hard: At least as hard as any NP problem
NP-Complete: Both NP and NP-Hard
Complexity Class Definitions
  • P: The class of decision problems that can be solved on a deterministic Turing machine in polynomial time ($\mathcal{O}(n^k)$).
  • NP: The class of decision problems whose solutions can be **verified** in polynomial time on a deterministic machine, or solved in polynomial time on a non-deterministic machine.
  • NP-Hard: A class of problems (not necessarily decision problems) to which any problem in NP can be reduced in polynomial time. Solving an NP-Hard problem in polynomial time would mean all NP problems could also be solved in polynomial time.
  • NP-Complete: The subset of problems in NP that are also NP-Hard. These are the absolute hardest decision problems within the NP class. Examples include 3-SAT, the Traveling Salesperson decision problem, Vertex Cover, and Independent Set.
Polynomial-Time Reduction Framework

To prove a new problem $Y$ is NP-Complete:

  1. Prove that $Y \in \text{NP}$ by showing a proposed solution can be verified in polynomial time.
  2. Select a known NP-Complete problem $X$ and reduce it to $Y$ in polynomial time ($X \le_P Y$). This proves that $Y$ is at least as hard as $X$, making it NP-Hard.
Critical Analytical Traps:
  • Reduction Direction Error: A common mistake is reversing the reduction direction. To prove problem $Y$ is NP-Hard, you must reduce a **known NP-Complete problem $X$ to your target problem $Y$** ($X \le_P Y$). Reducing $Y$ to $X$ ($Y \le_P X$) only proves that $Y$ is no harder than $X$, which does not establish NP-Hardness.
