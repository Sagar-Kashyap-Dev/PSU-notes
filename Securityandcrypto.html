The CIA Triad Architecture

The foundation of information security relies on balancing three distinct, complementary targets:

  • Confidentiality: Ensures that private assets remain hidden from unauthorized actors during transport and storage (enforced via symmetric/asymmetric encryption matrices).
  • Integrity: Guarantees that data hasn't been altered or corrupted by malicious actors or system failures (enforced using cryptographic hash signatures).
  • Availability: Ensures authorized users have uninterrupted access to critical systems and data assets (protected by load balancing, automated backups, and DDoS mitigation paths).

2. Symmetric Block Cryptography

Data Encryption Standard (DES) Architecture

DES is a historic symmetric block cipher that operates on 64-bit blocks using a 56-bit effective key size (with 8 parity bits dropped). It is built on a 16-round Feistel Network Structure:

In each round, the data block is split into 32-bit left ($L_i$) and right ($R_i$) halves, processing state changes via the following transformation rule:

$$L_{i+1} = R_i$$ $$R_{i+1} = L_i \oplus F(R_i, K_i)$$

Note: Due to its short 56-bit key length, DES is highly vulnerable to modern brute-force hardware sweeps and is considered obsolete.

Advanced Encryption Standard (AES) Architecture

AES is a robust modern block cipher that operates on fixed 128-bit data blocks. It supports variable key lengths (128, 192, or 256 bits) and replaces Feistel designs with a highly efficient Substitution-Permutation Network (SPN) matrix:

  1. SubBytes: Runs non-linear byte-by-byte substitutions using an established S-box lookup matrix to maximize confusion.
  2. ShiftRows: Performs cyclic left shifts on the last three rows of the 4x4 state array to diffuse patterns across blocks.
  3. MixColumns: Multiplies columns by an algebraic transformation matrix in a Galois Field ($GF(2^8)$) to add inter-byte dependencies.
  4. AddRoundKey: Mixes the specific round subkey directly into the current state block using a standard bitwise XOR ($\oplus$) operation.

3. Asymmetric Public-Key Cryptosystems

The RSA Algorithm Mathematical Engine

RSA derives its strength from the immense computational difficulty of factoring the product of two ultra-large prime numbers:

  • Key Pair Generation Steps:
    1. Select two distinct, hidden large prime integers, $p$ and $q$.
    2. Compute the public modulus product: $n = p \times q$.
    3. Calculate Euler's totient function: $\phi(n) = (p - 1)(q - 1)$.
    4. Choose an encryption exponent $e$ such that: $1 < e < \phi(n)$ and $\gcd(e, \phi(n)) = 1$.
    5. Compute the private decryption exponent $d$ using the modular multiplicative inverse: $$d \equiv e^{-1} \pmod{\phi(n)} \implies (d \times e) \equiv 1 \pmod{\phi(n)}$$
  • Cryptographic Execution Functions: $$\text{Encryption (Public Key } \{e, n\} \text{): } C = M^e \pmod n$$ $$\text{Decryption (Private Key } \{d, n\} \text{): } M = C^d \pmod n$$
Diffie-Hellman Key Exchange Protocol

Enables two independent parties to safely establish a shared secret key over an unencrypted, public communication channel without transmitting the key itself:

  1. Both parties openly agree on a large prime modulus $p$ and a primitive root generator base $g$.
  2. Alice chooses a secret private integer $a$, then computes and dispatches her public value to Bob: $$A = g^a \pmod p$$
  3. Bob chooses a secret private integer $b$, then computes and dispatches his public value to Alice: $$B = g^b \pmod p$$
  4. Both parties derive the identical shared symmetric secret key $K$ independently: $$\text{Alice: } K = B^a \pmod p = (g^b)^a \pmod p$$ $$\text{Bob: } K = A^b \pmod p = (g^a)^b \pmod p \implies K = g^{ab} \pmod p$$

4. Integrity, Signatures & Public Key Infrastructures

Cryptographic Hash Functions Engine (MD5 & SHA)

Hash functions convert arbitrary-length input data into fixed-size byte strings. A secure cryptographic hash must guarantee three core mathematical behaviors:

  • Pre-image Resistance: Given a hash output $H$, it is computationally impossible to reverse the operation to find the original input string $x$ such that $h(x) = H$.
  • Second Pre-image Resistance: Given a specific input $x_1$, it is computationally impossible to find a different input string $x_2$ that yields the exact same hash output: $h(x_1) = h(x_2)$.
  • Collision Resistance: It is computationally impossible to find *any* two arbitrary, distinct input strings $x_1$ and $x_2$ that produce matching hash outputs: $h(x_1) = h(x_2)$.

Algorithm Variants: MD5 outputs 128-bit digests but is broken due to collision vulnerabilities. Modern setups rely on SHA-256 (secure 256-bit digests) or SHA-3 architectures.

Digital Signatures & Digital Certificates
  • Digital Signatures: Provide Authentication, Integrity, and Non-repudiation. The sender runs their message through a hash function, encrypts the resulting digest with their *private key*, and appends this signature payload to the message. The recipient decrypts the signature using the sender's *public key* and compares the hash values to verify authenticity.
  • Digital Certificates (X.509): Solve identity spoofing by binding a public key directly to an individual or organization. Trusted third-party **Certificate Authorities (CAs)** digitally sign these identity bundles to confirm their validity.

5. Network Boundary Controls & Threat Detection

Firewall Taxonomy Classifications

Firewalls block unauthorized traffic by monitoring network boundaries at different layers of the OSI model:

  1. Packet Filters: Inspect individual packets independently at the Network layer, filtering traffic based on static source/destination IPs and port numbers.
  2. Stateful Inspection Firewalls: Track active connection states across a dedicated state table. They verify that incoming packets belong to an active, established handshake sequence.
  3. Application Firewalls (Proxies): Intercept and analyze traffic at the Application layer, inspecting actual payload content (e.g., parsing HTTP strings for malicious commands).

Detection Methodologies: Signature-Based (matches traffic against known attack patterns; cannot catch zero-day exploits) vs. Anomaly-Based (establishes a normal baseline and flags statistical deviations).

6. Advanced Cryptographic Verification

Challenge-Response Authentication Framework

Prevents replay attacks by ensuring raw password hashes are never transmitted across the network during authentication:

  1. A client requests access to a secure service.
  2. The server generates and returns a randomized, single-use string called a **Nonce** (the challenge token).
  3. The client combines this nonce with their shared secret password, hashes the combined string, and sends it back to the server (the response token).
  4. The server runs the exact same calculation independently. If the hashes match, access is granted.
Zero-Knowledge Proofs (ZKP)
  • An advanced protocol that lets a prover confirm to a verifier that they possess a specific secret value without revealing any information about the secret itself.
  • Relies on iterative cycles of mathematical queries, challenges, and proofs. This confirms ownership with high statistical certainty while keeping the underlying data completely hidden.
