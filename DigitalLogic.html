1. Number Systems & Complement Arithmetic

Radix (r): Base Weighting Value
(r-1)'s Comp: Diminished Radix Complement

Mathematical Complements Formula Definition

Given an $n$-digit integer $N$ encoded in a base-radix $r$ system:

  • Diminished Radix Complement ($(r-1)$'s Complement): $$\text{Complement} = (r^n - 1) - N$$ *(Evaluated by subtracting every individual digit of $N$ from the highest digit value $r-1$). Examples include **1's complement** ($r=2$) and **9's complement** ($r=10$).*
  • True Radix Complement ($r$'s Complement): $$\text{Complement} = r^n - N = \left[ (r^n - 1) - N \right] + 1$$ *(Obtained by adding 1 to the least significant bit of its corresponding $(r-1)$'s complement). Examples include **2's complement** ($r=2$) and **10's complement** ($r=10$).*

Critical Overflow Detection Rule:
  • An arithmetic overflow occurs when adding two numbers with the same sign produces a result with the opposite sign. In an Arithmetic Logic Unit (ALU), this condition is detected by checking the carry bits entering and leaving the sign column: $$\text{Overflow} = C_{\text{in}} \oplus C_{\text{out}}$$

XOR/XNOR Identity Matrix Theorems
  • $A \oplus 0 = A, \quad A \oplus 1 = \bar{A}, \quad A \oplus A = 0, \quad A \oplus \bar{A} = 1$
  • Both XOR and XNOR obey the commutative and associative laws: $$A \oplus (B \oplus C) = (A \oplus B) \oplus C$$
  • Parity Rule: For $n$ independent input variables, an XOR gate output evaluates to $1$ if and only if the input combination contains an **odd number of 1s**.

3. Boolean Algebra & Karnaugh Map Minimization

SOP: Sum of Products (ORing ANDs)
POS: Product of Sums (ANDing ORs)
Minterm (m): Product Term Evaluating to 1
Maxterm (M): Sum Term Evaluating to 0
Axiomatic Theorems of Boolean Algebra
  • De Morgan's Laws: $$\overline{A \cdot B} = \bar{A} + \bar{B}, \quad \overline{A + B} = \bar{A} \cdot \bar{B}$$
  • Consensus Theorem: $$A B + \bar{A} C + B C = A B + \bar{A} C$$
  • Distributive Law Expansion: $$A + (B \cdot C) = (A + B) \cdot (A + C)$$
Karnaugh Map (K-Map) Engine Architecture

K-Maps organize boolean truth tables into multi-dimensional grids mapped via **Gray Code sequences** (ensuring only a single bit changes between adjacent cells to maintain spatial adjacency properties).

  • Prime Implicant (PI): A product term obtained by combining adjacent cells in a K-Map into the largest possible group (subcube of size $2^k$). It cannot be further expanded.
  • Essential Prime Implicant (EPI): A Prime Implicant that contains at least one minterm cell that is not covered by any other Prime Implicant. EPIs **must always be included** in the final minimized boolean expression.

4. Combinational Circuits — Arithmetic Cores

Half Adder: 2 Inputs, 2 Outputs Profile
Full Adder: 3 Inputs, 2 Outputs Profile
Look-Ahead: Parallel Carry Generation
Comparator: Magnitude Difference Evaluator
Full Adder Logic Specifications

Given binary inputs $A$, $B$, and Carry-In $C_{\text{in}}$:

  • $$\text{Sum } (S) = A \oplus B \oplus C_{\text{in}}$$
  • $$\text{Carry-Out } (C_{\text{out}}) = A B + B C_{\text{in}} + A C_{\text{in}} = A B + C_{\text{in}}(A \oplus B)$$
  • *Implementation Limits:* A full adder can be realized using exactly **two half adders and a single OR gate**.
Carry Look-Ahead Adder (CLA) Engine

To bypass the long propagation delays found in standard Ripple-Carry Adders, a Carry Look-Ahead Adder computes carry bits in parallel using generate ($G_i$) and propagate ($P_i$) functions:

  • $$\text{Carry Generate: } G_i = A_i \cdot B_i, \quad \text{Carry Propagate: } P_i = A_i \oplus B_i$$
  • $$\text{Parallel Carry Equations: } C_{i+1} = G_i + P_i C_i$$ $$C_1 = G_0 + P_0 C_0$$ $$C_2 = G_1 + P_1 G_0 + P_1 P_0 C_0$$

5. Combinational Data Path Modules (MUX / DEMUX / Decoders)

MUX: $2^n$-to-1 Data Selector
DEMUX: 1-to-$2^n$ Data Distributor
Decoder: $n$-to-$2^n$ Minterm Generator
Encoder: $2^n$-to-$n$ Priority Binary Codec
Multiplexer (MUX) Synthesis Functions

A $2^n \times 1$ MUX routes one of its $2^n$ data inputs to a single output line based on the binary state of its $n$ selection lines. It acts as a universal function generator:

  • A $2^n \times 1$ MUX can implement **any boolean function containing up to $n+1$ independent variables** without requiring any additional external logic gates.
Decoders & Encoders Architecture
  • Decoder ($n \times 2^n$): Converts an $n$-bit binary input into an active signal at one of its $2^n$ unique output lines. The outputs correspond exactly to the set of **canonical minterms** for those input variables.
  • Priority Encoder: Resolves simultaneous input requests by converting the highest-priority active input line into its corresponding binary code. It includes a *Valid Output ($V$)* bit to distinguish an active input on line 0 from a state where no inputs are active.

Critical Race-Around Hazard Condition:
  • In a level-triggered JK flip-flop, a **Race-Around Condition** occurs when $J=K=1$ and the clock pulse width ($t_w$) is longer than the flip-flop's internal propagation delay ($\Delta t$). This causes the output to toggle repeatedly and unpredictably during a single clock cycle. This hazard is eliminated by using **edge-triggered clocking inputs** or a Master-Slave flip-flop architecture.

7. Registers & Shift Counter Configurations

SISO: Serial In / Serial Out Pipe
SIPO: Serial In / Parallel Out Pipe
Ring Counter: Straight Loop ($N$ Modulo States)
Johnson Counter: Inverted Loop ($2N$ Modulo States)
Shift Register Interconnect Topologies
  • SISO / SIPO / PISO / PIPO: Linear arrays of cascaded flip-flops used to store or reformat binary data. Moving an $n$-bit data word through a SISO register requires $n$ clock cycles for the write phase, and $n-1$ additional cycles to fully shift the data out.
Circulating Shift Counter State Metrics

Specialized shift registers configured with feedback loops loop back to the input stage:

  • Ring Counter: The primary output of the final flip-flop ($Q_n$) connects directly back to the first stage input ($D_0$). An $N$-bit shift register configured as a Ring Counter provides exactly **$N$ unique operational states**.
  • Johnson Counter (Twisted Ring Counter): The inverted output of the final stage ($\bar{Q}_n$) loops back to the first stage input ($D_0$). An $N$-bit register configured this way provides **$2N$ unique operational states**, doubling its counting capacity.

8. Asynchronous & Synchronous Counters

Ripple Counter: Cascaded Clock Propagations
Synchronous Counter: Unified Parallel Shared Clock
Mod-M Counter: M Unique Valid Counting States
Max Frequency: $f_{\text{max}} = 1 / (n \cdot t_{\text{pd}})$ Limit
Counter Architectures Comparison
  • Asynchronous (Ripple) Counter: The clock signal drives only the first flip-flop stage. Subsequent stages are clocked by the output of the preceding flip-flop. This causes a delayed "ripple" effect across the counter.
    The maximum operating frequency is limited by the accumulated propagation delays across all $n$ stages: $$f_{\text{max}} = \frac{1}{n \cdot t_{\text{pd}}}$$
  • Synchronous Counter: A unified, parallel clock signal **drives all flip-flops simultaneously**. This eliminates accumulated propagation delay, allowing the counter to run at much higher frequencies. The maximum frequency is bounded only by the delay of a single stage plus its look-ahead logic gates: $$f_{\text{max}} = \frac{1}{t_{\text{pd}} + t_{\text{gate}}}$$
State Design Analysis Sequence

To construct a synchronous sequence counter:

  1. Derive the required State Transition Diagram and State Table.
  2. Determine the next-state values based on the flip-flop excitation rules.
  3. Minimize the input control expressions using K-Maps.
  4. Implement the look-ahead combinational logic circuits.
