CIL MT Systems Last-Week DSA Revision

Persistent Checklist: Tracking via LocalStorage

1. Array

Facts / Points
  • Contiguous memory allocation.
  • Supports random access.
  • Fixed size.
  • Insertion/deletion in middle is costly.
  • Binary search works only on sorted arrays.
  • Cache friendly.
Formulas & Complexities
Address calculation:
LOC(A[i]) = Base + (i - LB) × w
AccessO(1)
Linear SearchO(n)
Binary SearchO(log n)

2. Linked List

Facts / Points
  • Dynamic size.
  • Non-contiguous memory.
  • Random access not possible.
  • Extra memory needed for pointer(s).
  • Easy insertion/deletion.
Formulas & Complexities
SearchO(n)
Insert at StartO(1)
Delete at StartO(1)

3. Stack

Facts / Points
  • LIFO (Last In First Out).
  • Insert/Delete from one end (TOP).
  • Used in recursion, expression evaluation, and parenthesis matching.
Formulas & Conditions
Overflow Condition: TOP == MAX - 1
Underflow Condition: TOP == -1
PushO(1)
PopO(1)
SearchO(n)

4. Queue

Facts / Points
  • FIFO (First In First Out).
  • Insert at Rear, Delete at Front.
  • Used in operating system scheduling and BFS (Breadth-First Search).
Formulas & Conditions
Circular Queue Full Condition:
(REAR + 1) % N == FRONT
EnqueueO(1)
DequeueO(1)

5. Tree

Facts / Points
  • Hierarchical structure.
  • Root node has no parent.
  • Tree is connected and acyclic.
  • n nodes have n − 1 edges.
Formulas
Total Edges = n - 1
Leaf nodes in full binary tree: L = I + 1   (where I = internal nodes)

6. Binary Tree

Facts / Points
  • Each node has at most 2 children.
  • Types: Full, Complete, Perfect, Balanced, Skewed.
Formulas Worth Memorizing
Maximum nodes at level k = 2k
Maximum nodes for height h = 2h+1 - 1
Minimum nodes for height h = h + 1
Leaf nodes in perfect tree = 2h
Internal nodes in perfect tree = 2h - 1
Total nodes from leaf nodes: N = 2L - 1
Null links in binary tree with n nodes = n + 1

7. Tree Traversal

Facts / Points
  • Preorder: Root → Left → Right
  • Inorder: Left → Root → Right
  • Postorder: Left → Right → Root
  • Level Order: Breadth-First Search (BFS)
Complexity
All TraversalsO(n)

8. Binary Search Tree (BST)

Facts / Points
  • Left subtree < Root
  • Right subtree > Root
  • Inorder traversal always gives sorted order.
  • Not necessarily balanced.
Time Complexities (Search / Insert / Delete)
Balanced Best CaseO(log n)
Skewed Worst CaseO(n)

9. AVL Tree

Facts / Points
  • Self-balancing BST.
  • Rotations to balance: LL, RR, LR, RL.
Formulas
Balance Factor (BF) = Height(Left) - Height(Right)
Valid AVL condition: -1 ≤ BF ≤ 1
All Operations (Best/Worst)O(log n)

10. Heap

Facts / Points
  • Complete Binary Tree structure.
  • Min Heap: Parent ≤ Children | Max Heap: Parent ≥ Children.
  • Not a BST.
Formulas (Array representation index-0)
Parent(i) = ⌊ (i-1)/2 ⌋
Left Child(i) = 2i + 1  |  Right Child(i) = 2i + 2
Build HeapO(n)
Insert / DeleteO(log n)
Heap SortO(n log n)

11. Hash Table

Facts / Points
  • Uses a hash function to map keys to slots.
  • Average search is exceptionally fast. Collision is possible.
  • Collision handling: Chaining (Open), Linear/Quadratic Probing, Double Hashing (Closed).
Formulas
Load Factor (α) = n / m   (where n = items, m = slots)
Average SearchO(1)
Worst Case SearchO(n)

12. Graph

Facts / Points
  • Collection of vertices (V) and edges (E).
  • Directed or Undirected. A tree is a connected acyclic graph.
Formulas
Undirected Handshaking: ∑ Degree = 2E
Max edges (Undirected) = n(n-1) / 2
Max edges (Directed) = n(n-1)

13. BFS

Facts / Points
  • Uses Queue.
  • Level-by-level traversal.
  • Finds shortest path in an unweighted graph.
Time ComplexityO(V + E)
Space ComplexityO(V)

14. DFS

Facts / Points
  • Uses Stack.
  • Goes deep down alternative paths first.
  • Used in cycle detection and topological sorting.
Time ComplexityO(V + E)
Space ComplexityO(V)

15. Sorting Algorithms

Key Sorting Parameters
  • Bubble Sort: Stable, In-place. Best: O(n), Worst: O(n2)
  • Selection Sort: Minimum swaps, Not stable. Always: O(n2)
  • Insertion Sort: Adaptive (Excellent for nearly sorted). Stable. Best: O(n), Worst: O(n2)
  • Merge Sort: Divide & Conquer, Stable. Time: O(n log n), Space: O(n)
  • Quick Sort: Divide & Conquer, In-place, Pivot-based. Avg: O(n log n), Worst: O(n2)
  • Heap Sort: Uses Heap, Not stable, In-place. Always: O(n log n)

16. Searching

Quick Compares
  • Linear Search: Works on unsorted data. Complexity: O(n)
  • Binary Search: Strictly requires sorted data. Divide & Conquer. Complexity: O(log n)
Formula
Maximum comparisons in Binary Search = ⌊ log2 n ⌋ + 1

17. Dynamic Programming

Facts / Points
  • Overlapping subproblems & Optimal substructure.
  • Memoization = Top-Down approach.
  • Tabulation = Bottom-Up approach.
Typical State Equation
DP[i] = optimal_choice(solution using previous sub-states)

18. Greedy Algorithm

Facts / Points
  • Chooses locally optimal solution at each step.
  • Does not guarantee a global optimum for all problems.
  • Core Applications: Kruskal's, Prim's, Huffman Coding, Dijkstra's.

19. Minimum Spanning Tree (MST)

Facts / Points
  • Connects all vertices with minimum possible total edge weight. No cycles.
  • Algorithms: Prim's (vertex-centric) & Kruskal's (edge-centric).
Formula
Total edges in any MST = n - 1

20. Shortest Path Algorithms

Core Comparisons
  • Dijkstra's: Greedy. Does not allow negative edge weights. Complexity with min-heap: O((V + E) log V)
  • Bellman-Ford: Dynamic Programming. Allows negative weights and detects negative cycles. Complexity: O(V × E)

⚠ Critical CIL MT One-Line MCQ Traps

  • BST ≠ Balanced Tree (unless stated as AVL/Red-Black).
  • Heap ≠ BST (Heap satisfies structural and heap properties, not order properties).
  • BFS → Queue | DFS → Stack.
  • Inorder(BST) Always → Sorted Ascending Order.
  • Complete Tree ≠ Full Tree.
  • Merge Sort → Stable | Quick Sort → Worst Case O(n2).
  • Dijkstra → No Negative Edge | Bellman-Ford → Negative Edges Allowed.
  • Hashing Average Search Complexity → O(1).
  • Binary Search → Sorted Data Array Required.
  • Tree with n nodes → Exactly n − 1 edges.
  • Full Binary Tree → L = I + 1.
  • Heap is always structurally a Complete Binary Tree.
