1. TOC Basics

Facts / Points
  • Theory of Computation explores the fundamental capabilities and boundaries of math problems solvable using distinct conceptual mathematical machines.
  • Core Compilers / Compute Modules: Finite Automata → Pushdown Automata → Linear Bounded Automata → Turing Machines.
Common MCQ Traps
  • Regular Languages: Represent the absolute simplest computational language class footprint.
  • Turing Machine: Remains the absolute most powerful computational model archetype (maps to general computing capacity limits).

2. Alphabet, String & Language Definitions

Foundations
  • Alphabet (Σ): A strict, non-empty **finite set** of atomic symbols.
  • String: A specific finite linear sequence of symbols derived explicitly from an input alphabet set.
  • Language (L): A set of legal strings structured over a target alphabet domain. Can be finite or infinite.
Common MCQs
  • Mathematical Distinction Trap: Ø ≠ {ε}
  • Ø represents a completely empty language set containing zero elements.
  • {ε} represents a language set containing exactly one item: the **empty string** (length zero).

3. Finite Automata (FA) Core Frameworks

Automata Variations
  • DFA (Deterministic Finite Automata): Has exactly one predictable transition path execution path per input symbol from any state.
  • NFA (Nondeterministic Finite Automata): Can process zero, one, or multiple dynamic transition paths simultaneously for a single symbol state step.
Common MCQs
  • DFA structure implies zero structural ambiguity or choice variation.
  • Equivalence Trap: NFA and DFA possess identical mathematical language recognition power. Any language parsed by an NFA can be parsed by a corresponding DFA via subset construction.

4. Deterministic Finite Automata (DFA)

The Standard 5-Tuple Definition
M = (Q, Σ, δ, q0, F)
  • Q: Finite non-empty set of internal states.
  • Σ: Finite input alphabet set.
  • δ: Transition map function operating as: δ : Q × Σ → Q.
  • q0: Explicit initial start state member where q0 ∈ Q.
  • F: Set of final/accepting states where F ⊆ Q.
Common MCQs
  • A DFA always contains exactly **one single initial start state**.
  • String acceptance is evaluated strictly by verifying if processing finishes inside an active **final state**.

5. Nondeterministic Finite Automata (NFA)

Operational Shifts
  • Transition maps return a powerset array block: δ : Q × Σ → 2Q.
  • Can support spontaneous state modifications using empty ε-moves (epsilon jumps) without consuming an active input symbol.
Common MCQs
  • NFA: Substantially simpler to conceptually model and design for specific word target lookups.
  • DFA: Drastically simpler to write, convert, and implement as an active execution code algorithm.

6. Regular Expressions (RE)

Formal Operational Operators
  • + or | → Represents boolean Union / OR operations.
  • . → Coordinates structural syntax Concatenation properties.
  • * → Kleene Star operator (represents zero or more repetitions of the target sequence).
Common MCQs Syntax
  • The universal expression (a + b)* compiles out into every conceivable string permutation sequence composed entirely of any count of 'a' and 'b' elements, including the zero length string ε.

7. Regular Languages & Closure Properties

Definitions
  • A language is formally classified as regular if and only if it can be verified via a valid DFA, NFA, or parsed by a descriptive Regular Expression syntax mapping.
Closure Matrix
  • Regular languages are highly stable and are explicitly CLOSED under: Union, Intersection, Complement, Set Difference, Kleene Star, and Concatenation.

8. Pumping Lemma (Regular Proofs)

The Operational Condition Criteria
If L is regular, any string s with length |s| ≥ p can be partitioned into s = xyz satisfying:
1. |xy| ≤ p
2. |y| > 0
3. xynz ∈ L for every single iteration integer n ≥ 0
Common MCQs Core Logic Trap
  • The Pumping Lemma can **NEVER** be utilized to prove a language is regular. It is applied exclusively as an inductive contradiction weapon to prove that a language is NOT regular (e.g., proving L = anbn lacks regular status due to infinite memory constraint traps).

9. Context Free Grammars (CFG)

The Formal 4-Tuple Architecture
G = (V, Σ, P, S)
  • V: Finite set of non-terminal variables.
  • Σ: Finite set of terminal characters completely disjointed from V.
  • P: Production rules mapped mapping down as: A → α where A ∈ V and α ∈ (V ∪ Σ)*.
  • S: Unique designated top-level initial start symbol where S ∈ V.

10. Context Free Languages (CFL)

Structural Properties
  • Languages generated natively by valid Context Free Grammars. Can be completely parsed mechanically via a standard stack-backed **Pushdown Automaton (PDA)**.
Common MCQs Examples
  • Nested structural matching problems, balance loops, or standard arithmetic balanced parentheses strings (e.g., L = { anbn | n ≥ 0 }) constitute standard examples of **Context Free Languages**.

11. Pushdown Automata (PDA) Architecture

Structural Hardware Engine
PDA = Finite State Control Engine + 1 Infinite LIFO Memory Stack
  • Provides the structural execution environment for Context Free Language analysis. Stack tracks operations via standard **Push (insert)** and **Pop (evacuate/evaluate)** actions.
Common MCQs Limits
  • The introduction of a stack gives the PDA the ability to manage **nested structural dependencies** and infinite count checks.
  • Deterministic vs Non-Deterministic Power Trap: Unlike Finite Automata, Non-Deterministic PDAs (NPDA) are strictly more powerful than Deterministic PDAs (DPDA). DPDAs cannot recognize all context-free languages (e.g., even palindromes like wwR).

12. PDA Acceptance Standards

Acceptance Frameworks
  • Acceptance by Final State: Reaches a verified matching accepting state block after emptying the input string.
  • Acceptance by Empty Stack: Empties the internal LIFO memory execution stack completely to signal successful validation.
Common MCQs Identity
  • Both acceptance design methods are mathematically equivalent. Any language validated via final state parameters can be modified to validate via empty stack configurations.

13. Pumping Lemma (CFL Proofs)

The Operational 5-way Split Split Criteria
If L is CFL, any string s with length |s| ≥ p can be partitioned into s = uvwxy satisfying:
1. |vwx| ≤ p
2. |vx| > 0
3. uvnwxny ∈ L for all n ≥ 0
  • Used to prove a target language layout is NOT Context-Free (e.g., proving triple system alignments like L = anbncn lie outside CFL capabilities).

14. Turing Machine (TM) Foundations

Structural Properties
  • The absolute peak computational capability reference design archetype. Maps down as the definitive mathematical model of general universal computation.
  • Physical Concepts: An active read/write R/W Head scanning across a logically **Infinite Storage Tape** partitioned into distinct address cells.
Common MCQs Mapping
  • If an algorithmic procedure can handle a problem on any digital supercomputer, a **Turing Machine** can process it.

15. Turing Machine 7-Tuple Architecture

M = (Q, Σ, Γ, δ, q0, B, F)
  • Q: Finite set of operational internal state maps.
  • Σ: Input alphabet set (Symbols allowed inside initial raw input strings).
  • Γ: Complete Tape Alphabet space (where Σ &subf; Γ).
  • δ: Next step transition map function: δ : Q × Γ → Q × Γ × {L, R} (Moves left or right).
  • q0: Unique initial start state element.
  • B: Explicit **Blank Symbol** placeholder indicating unallocated tape space (where B ∈ Γ but B ∉ Σ).
  • F: Set of designated final accepting states.

16. Turing Machine Variations

Structural Variants
  • Multi-tape Turing Machines (uses separate parallel track heads).
  • Multi-track Turing Machines.
  • Non-deterministic Turing Machines (NTM).
Common MCQs Equivalency Principle
  • Power Trap: All structural multi-tape or non-deterministic Turing Machine variants are **completely equivalent in absolute computing power** to a standard single-tape deterministic Turing Machine. They only modify performance execution speed metrics, not language coverage capabilities.

17. Decidability & Undecidability

Core Class Definitions
  • Decidable Problems: Problems that can be evaluated via an explicit algorithm that is guaranteed to finish and **halt for every input** string (either accepting or rejecting safely).
  • Undecidable Problems: Problems for which it is mathematically impossible to construct a universal foolproof algorithm that guarantees a clean halt decision output for every input scenario.

18. The Halting Problem

Mathematical Fact
  • It is impossible to create a generalized algorithm that takes an arbitrary computer program file and an input, and correctly predicts whether that program will run to completion or loop infinitely.
Common MCQs Identity
  • The **Halting Problem** is the classic, definitive proof of an **undecidable problem**, proven by Alan Turing using diagonalization methods.

19. Recursive vs. Recursively Enumerable (RE)

Language Properties
  • Recursive Languages (Decidable): Parsed via a Turing Machine that **always halts** without exception, whether a string is accepted or rejected.
  • Recursively Enumerable (RE) Languages (Semi-Decidable): Parsed via a Turing Machine that completes and halts with an 'Accept' flag if the string is in the language. However, if the string is invalid, the machine may loop infinitely.
Common MCQs Complement Trap
  • If a language L is Recursively Enumerable (RE) **AND** its complement L' is also Recursively Enumerable, then the language L is guaranteed to be **Recursive (Decidable)**.

22. Ambiguity in Grammar Layouts

Facts / Points
  • A grammar is formally classified as ambiguous if there exists at least one valid string that can produce **two or more distinct parse trees** (or separate leftmost/rightmost derivations).
  • Inherent Ambiguity: A context-free language is inherently ambiguous if every possible CFG that generates it is guaranteed to be ambiguous.
Common MCQs Compiler Core Trap
  • Ambiguous grammars cause structural parsing issues in compilers because they lead to multiple competing interpretation paths for a single line of code.

23. Parsing Techniques

Architectures
  • Top-Down Parsers: Attempts to reconstruct the target parse tree starting from the root start symbol downwards (e.g., LL(1), Recursive Descent parsers).
  • Bottom-Up Parsers: Begins parsing at the raw terminal strings and works backwards towards the start symbol via string reduction operations (e.g., LR(k), Shift-Reduce parsers).
Common MCQs Power Comparison
  • **Bottom-Up Parsing (LR frameworks)** is strictly more powerful and supports a broader range of language grammars than Top-Down Parsing (LL frameworks).

24. Grammar Normal Forms (CNF & GNF)

CNF (Chomsky Normal Form) Structural Rules
All production rules must match exactly one of these two formats:
1. A → BC  (Variable leads to exactly two variables)
2. A → a   (Variable leads to exactly one terminal character)
GNF (Greibach Normal Form) Structural Rules
All production rules must match this format structure:
A → aα  (Variable leads to exactly one terminal followed by zero or more variables)
Common MCQs Bounds
  • Converting a standard CFG into CNF yields a parse tree where every internal step node has a maximum branching factor of exactly 2 (a binary tree framework).

⚠ Critical CIL MT TOC MCQ Traps Checklist

  • DFA transitions are completely deterministic; NFA transitions support multiple paths. They have **identical mathematical power**.
  • Regular Expressions correspond directly to Finite Automata. Context-Free Grammars require Pushdown Automata.
  • Turing Machines represent the peak computational capability in the Chomsky hierarchy.
  • The Pumping Lemma is an inductive tool used to prove that a language is NOT regular/context-free; it cannot verify regular status.
  • Pushdown Automata utilize an isolated LIFO memory stack. Turing Machines run across an infinite R/W tape.
  • The Halting Problem is mathematically **undecidable**.
  • Recursive Languages are completely decidable and are guaranteed to halt. Recursively Enumerable languages can loop infinitely on invalid inputs.
  • Regular languages are closed under complementation operations. Context-Free Languages (CFL) are **NOT closed under complementation or intersection**.
  • Ambiguity means a grammar can generate **two or more distinct parse trees** for a single string.
  • Finite Automata maintain zero tracking memory records; Pushdown Automata contain stack memory storage links.
