1. Operating System Basics

Facts / Points
  • Operating System acts as an interface between user and hardware.
  • Manages resources: CPU, Memory, I/O Devices, Files.
  • Provides convenience and efficiency.
Functions
  • Process Management
  • Memory Management
  • File Management
  • Device Management
  • Security
Common MCQ Traps
  • Kernel = core part of OS.
  • OS is a resource manager.
  • BIOS is NOT an operating system.
Formulas

None.

2. Types of Operating Systems

Facts / Points
  • Batch OS: Jobs executed in batches. No user interaction.
  • Time Sharing OS: Multiple users share CPU.
  • Multiprogramming OS: Multiple programs in memory simultaneously.
  • Multitasking OS: Multiple tasks executed seemingly at the same time.
  • Real-Time OS (RTOS): Strict timing constraints.
Common MCQs
  • Hard RTOS: Missing deadline = complete system failure.
  • Soft RTOS: Occasional deadline miss is acceptable.
Formulas

None.

3. Operating System Structure

Facts / Points
  • Monolithic Kernel: Entire OS works in kernel mode.
  • Microkernel: Minimal services in kernel. Moves services to user space.
  • Layered OS: Organized cleanly in layers from hardware up to user level.
Common MCQs
  • Microkernel advantages: Better reliability and easier maintenance.
Formulas

None.

4. Processes

Facts / Points
  • Process: Program in execution.
  • Components: Code, Data, Stack, Heap, and Program Counter.
  • Process States: New, Ready, Running, Waiting (Blocked), Terminated.
Common MCQs
  • PCB (Process Control Block) stores: Process ID, State, CPU Registers, and Scheduling Information.
Formulas

None.

5. Threads

Facts / Points
  • Thread: Lightweight process (LWP).
  • Shares: Code, Data, and Files.
  • Owns: Stack and Registers.
  • Types: User-Level Threads and Kernel-Level Threads.
Common MCQs
  • Thread switching is significantly faster than process switching.
Formulas

None.

6. CPU Scheduling

Core Algorithms
  • FCFS (First Come First Serve): Non-preemptive, simple. Trap: Convoy Effect.
  • SJF (Shortest Job First): Provides minimum average waiting time. Trap: Long jobs may starve.
  • Priority Scheduling: Higher priority executes first. Trap: Starvation possible. Solution: Aging.
  • Round Robin: Time Quantum used. Preemptive. Trap: Very large quantum ≈ FCFS.
Common Formulas
Turnaround Time (TAT) = Completion Time − Arrival Time
Waiting Time (WT) = TAT − Burst Time
Response Time = First CPU Allocation − Arrival Time

7. Process Synchronization

Facts / Points
  • Critical Section: Shared resource access code segment area.
  • Core Requirements: Mutual Exclusion, Progress, Bounded Waiting.
Common MCQs
  • Race Condition: Multiple processes access and manipulate shared data concurrently.

8. Semaphore

Facts / Points
  • Semaphore: Integer synchronization variable.
  • Atomic Operations: Wait (P) and Signal (V).
  • Types:
    • Binary Semaphore (0 or 1 value only)
    • Counting Semaphore (Can take multiple instances/unbounded values)
Common MCQs
  • Binary semaphore behaves structurally similar to a mutex.

9. Mutex

Facts / Points
  • Mutex: Locking mechanism used to synchronize access to a resource (Mutual Exclusion Lock).
Common MCQs
  • Ownership Principle: Only the thread/owner that locks a mutex can unlock it. Semaphore does NOT have an ownership concept.

10. Deadlock

Facts / Points
  • Deadlock: A set of processes wait forever for events/resources held by each other.
Necessary Conditions (Coffman Conditions)
  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait
Common MCQs
  • Simultaneity: All four conditions MUST occur simultaneously for a deadlock to happen.

11. Deadlock Handling Methods

Methods
  • Deadlock Prevention: Restrict resource requests to break at least one Coffman condition.
  • Deadlock Avoidance: Uses dynamic tracking check. Requires safe state evaluation.
  • Deadlock Detection & Recovery: Let deadlock occur, detect it via graphs, then recover.
Common MCQs
  • Banker's Algorithm: Purely a deadlock avoidance algorithm.

12. Banker's Algorithm

Facts / Points
  • Used dynamically to determine: Safe State and Safe Sequence.
Common MCQs
  • Safe State: System is guaranteed to avoid a deadlock.
  • Unsafe State: Not necessarily deadlocked, but has the risk of falling into one.

13. Memory Management

Facts / Points
  • Logical Address: Virtual address generated by the CPU.
  • Physical Address: Actual physical location address inside RAM.
Common MCQs
  • MMU (Memory Management Unit): Hardware device that converts logical address to physical address at runtime.

14. Contiguous Memory Allocation

Types
  • Fixed Partitioning: Memory divided beforehand into static boundaries.
  • Dynamic Partitioning: Partitions created on-demand dynamically.
Common MCQs
  • External Fragmentation occurs explicitly in **dynamic partitioning**.

15. Fragmentation

Types
  • Internal Fragmentation: Wasted unassigned space inside an allocated memory block.
  • External Fragmentation: Total free memory exists to satisfy a request, but it is scattered non-contiguously.
Common MCQs
  • Paging completely removes **external fragmentation**.

16. Paging

Facts / Points
  • Page: Fixed-size logical block of a process.
  • Frame: Fixed-size physical block of physical memory.
Address Translation Split
Logical Address Structure: [ Page Number (d) | Offset (d) ]
Common MCQs
  • Paging eliminates external fragmentation, but may still suffer from **internal fragmentation** (on the last page).

17. Page Table

Facts / Points
  • Maps Page Number → Frame Number.
Common MCQs
  • TLB (Translation Lookaside Buffer): A high-speed hardware cache used specifically to store page table entries and speed up address translation.

18. Virtual Memory

Facts / Points
  • Allows execution of processes whose total space requirements are larger than physical RAM.
  • Implemented commonly using **Demand Paging**.
Common MCQs
  • Under demand paging, pages are only loaded into main memory when explicitly required.

19. Page Replacement Algorithms

Key Parameters
  • FIFO (First In First Out): Oldest page removed. Trap: Vulnerable to **Belady's Anomaly**.
  • LRU (Least Recently Used): Removes page unreferenced for the longest time. Generally better than FIFO.
  • Optimal: Removes page used farthest in the future. Acts as a theoretical benchmark.
Common MCQs
  • Belady's Anomaly: Increasing frame allocation leads to *more* page faults. Seen in FIFO, but **not** in LRU.

20. Thrashing

Facts / Points
  • A system is thrashing when it spends more time swapping pages in and out than executing actual instructions.
  • Cause: Excessive page faults due to overallocation.
  • Solutions: Increase frame availability, or reduce the degree of multiprogramming.

21. File Systems

Facts / Points
  • File: A named logical collection of related data.
  • Operations: Create, Read, Write, Delete.
Common MCQs
  • Directory mapping structures provide structural file organization.

22. Disk Scheduling

Algorithms
  • FCFS: Requests served in strict arrival order.
  • SSTF (Shortest Seek Time First): Picks request closest to head. Trap: May cause starvation.
  • SCAN: Head moves completely end-to-end (**Elevator algorithm**).
  • C-SCAN: Circular SCAN. Provides a more uniform waiting time by returning straight to the start.
Common MCQs
  • SSTF can cause starvation. SCAN or C-SCAN give better overall turnaround performance.

23. I/O Management

Facts / Points
  • Interrupt: Asynchronous signal sent to CPU by hardware to indicate an event.
  • DMA (Direct Memory Access): Device transfers whole data blocks directly to/from memory without continuous CPU involvement.
Common MCQs
  • DMA dramatically improves I/O efficiency by freeing up CPU cycles.

24. System Calls

Facts / Points
  • Provides an operational interface between a user program and the OS kernel.
  • Types: Process Control, File Management, Device Management, Information Maintenance.
Common MCQs
  • fork() creates a brand new duplicate child process in UNIX environments.

25. UNIX/Linux Basics

Common Commands
  • ls → List directory files
  • pwd → Output present working directory path
  • mkdir → Create a blank directory
  • rm → Remove file
  • cp / mv → Copy file / Move or rename file
Common MCQs
  • UNIX is explicitly built as a robust multiuser, multitasking operating system.

26. OS Formulas Worth Memorizing

Metrics
CPU Utilization = (CPU Busy Time / Total Time) × 100
Throughput = Number of Completed Processes / Unit Time
Paging Address Math
Effective Access Time (EAT) = (1 - p) × MAT + p × PFST

Where:
p = Page Fault Rate
MAT = Memory Access Time
PFST = Page Fault Service Time
