1. Number Systems & Deep Conversions

Radix Rules
  • Binary (Base 2): Digits {0, 1}. Radix bits tracking weights match powers of 2.
  • Octal (Base 8): Digits {0-7}. Requires grouping binary bits into clean packets of 3 starting from the binary point.
  • Hexadecimal (Base 16): Digits {0-9, A-F}. Requires grouping binary bits into clean packets of 4 bits.
Positional Parsing Representation
(1011)₂ = (1 × 2³) + (0 × 2²) + (1 × 2¹) + (1 × 2⁰) = 8 + 0 + 2 + 1 = (11)₁₀
Complements & Radix Arithmetic
  • 1's Complement: Bitwise inversion (Logical NOT applied across all individual binary positions).
  • 2's Complement: Created directly by adding 1 to the Least Significant Bit (LSB) of the 1's complement form.
    [2's Comp = 1's Comp + 1]
Common MCQ Traps
  • Modern processors utilize 2's complement exclusively for signed arithmetic subtractions to avoid the hardware complication of a dual zero value (+0 and -0).
  • Range Asymmetry: 2's complement exhibits asymmetric distribution limits. Because 0 is pooled directly with positive values, there is exactly one more negative number than positive numbers available in an n-bit allocation space.

2. Signed Number Representations

Representation Schemes
  • Signed Magnitude: MSB acts strictly as an isolated isolated sign flag (0 = positive, 1 = negative). The remaining (n-1) bits store raw absolute magnitude. Features a dual representation for zero (+0 and -0).
  • 1's Complement Scheme: Negative numbers are generated by cleanly inverting the absolute positive bit matrix. Also suffers from dual-zero redundancy bugs.
  • 2's Complement Scheme ⭐: The absolute hardware standard. Unifies zero to a single binary string, simplifying ALU addition/subtraction circuits.
Mathematical n-bit Boundary Limits
Representation Type Valid Structural Range Equation (n-bits)
Signed Magnitude -(2n-1 - 1) to +(2n-1 - 1)
1's Complement -(2n-1 - 1) to +(2n-1 - 1)
2's Complement ⭐ -2n-1 to +(2n-1 - 1)
High-Frequency Exam Check
  • For an 8-bit sign calculation track, the valid 2's complement capacity spans seamlessly from -128 to +127.

3. Boolean Algebra Theorems & Duality

Core Identities
  • Idempotent Law: A + A = A  |  A • A = A
  • Absorption Theorem: A + AB = A  |  A(A + B) = A
  • Consensus Theorem (Highly Asked): AB + A'C + BC = AB + A'C
The Duality Principle Ruleset
  • Any valid algebraic statement remains true if you perform the following swap adjustments throughout the equation:
    • Swap operators: + (OR) ↔ • (AND)
    • Swap identity units: 0 ↔ 1
    Warning: Do NOT change or invert variable sign components (leave A and A' completely untouched).

4. Logic Gates & XOR Properties

Primary Universal Equations
  • NAND Gate: Y = (AB)' → Output drops to 0 if and only if both inputs scale to 1.
  • NOR Gate: Y = (A + B)' → Output reaches 1 if and only if all inputs rest at 0.
XOR & XNOR Operators (Critical Focus)
  • XOR (Odd Function): Y = A ⊕ B = AB' + A'B. Fires high (1) if the input combination features an **odd number of 1s**. For 2 inputs, it drives 1 when inputs differ.
  • XNOR (Equivalence Gate): Y = A ⊙ B = AB + A'B'. Fires high (1) when inputs match.

6. Canonical Forms (SOP & POS)

SOP (Sum of Products)
  • Constructed using Boolean Minterms. Minterms map directly to input combinations that evaluate to a functional 1 output. Written using Sigma notation: F = Σ m(1, 3, 5).
POS (Product of Sums)
  • Constructed using Boolean Maxterms. Maxterms map directly to input combinations that evaluate to a functional 0 output. Written using Pi notation: F = Π M(0, 2, 4).
Combinatorics Rules
  • For an expression built with n separate boolean variables, the total possible unique minterm or maxterm combination instances is mathematically exactly 2ⁿ.

7. Karnaugh Maps (K-Map) Mechanics

Grid Configuration Track
  • 2 Variables → 4 Cells  |  3 Variables → 8 Cells  |  4 Variables → 16 Cells.
  • Cells are strictly arranged in **Gray Code sequence order** (00, 01, 11, 10). This ensuring adjacent cells differ by exactly **1 bit position**.
Essential Grouping Requirements
  • Groups must be sized exactly as powers of 2: **1, 2, 4, 8, or 16** cells.
  • Wrap-Around Allowed: Edges are logically continuous; the leftmost column wraps around to touch the rightmost column, and the top row wraps around to touch the bottom row.
  • Don't Care Conditions (X): Can be treated dynamically as 0 or 1. Use them only if they help expand an active group size (e.g., turning a pair into a quad), which minimizes the final term. Never make a standalone group consisting only of 'Don't Care' cells.

8. Combinational Circuit Building Blocks

Subtractor Architectures
  • Half Subtractor: Computes the difference of two bits. Equations:
    Difference = A ⊕ B  |  Borrow = A'B.
  • Full Subtractor: Computes the difference of two bits while incorporating a preceding stage borrow-in parameter (Bᵢ). Equations:
    Difference = A ⊕ B ⊕ Bᵢ  |  Borrow = A'B + (A ⊕ B)'Bᵢ.
Magnitude Comparators
  • Takes two binary inputs (A, B) and generates exactly one high flag output among three conditional lines: A > B, A = B, or A < B.
  • For an A = B condition across bits, it utilizes systematic cascading **XNOR** comparison checks.
Parity Generators
  • An error-detection hardware layout built by stacking cascading lines of **XOR** gates. Even parity forces the total bit stream 1-count to be even; odd parity forces it to be odd.

9. Adder Paradigms & Performance

Ripple Carry Adder (RCA)
  • Constructed by cascading n Full Adders in series. It is inherently slow because the carry bit must physically ripple step-by-step from the least significant full adder stage down to the most significant stage. Worse-case delay scales linearly: O(n).
Carry Lookahead Adder (CLA) ⭐
  • Drastically accelerates addition speeds by computing carry bits ahead of time in parallel using dedicated combinatorial logic equations. It determines carries using Carry Generate (Gᵢ = AᵢBᵢ) and Carry Propagate (Pᵢ = Aᵢ ⊕ Bᵢ) terms. Worst-case gate delay is decoupled from bit count and remains constant: O(1).

10. Sequential Circuits: Latches vs. Flip-Flops

Core Distinctions
  • Sequential logic relies on an active **feedback loop network** to establish tracking memory states. Output states depend on both current inputs and past execution values.
Core Design Parameter Latch Architecture Flip-Flop Architecture
Trigger Modality Level Triggered (Outputs track inputs continuously while the clock signal stays high/low) Edge Triggered (Outputs transition only during the precise rising or falling clock transition)
Hardware Complexity Low (Built from basic cross-coupled gates) Higher (Typically engineered using master-slave latch configurations)
Race Vulnerability Highly susceptible to state race conditions Highly robust against timing race conditions

11. Flip-Flop Characteristic Operations

Characteristic Equations Track
  • SR Flip-Flop Invalidation: When inputs are set to S = 1, R = 1, the outputs try to drive low simultaneously. This breaks the complementary requirement (Q = Q'), creating an **unpredictable/invalid state** upon clock release.
  • JK Flip-Flop: Solves the SR invalidation problem. When J = 1, K = 1, the state cleanly **toggles** on every active clock edge:
    Q_next = Q'.
  • D (Data) Flip-Flop: Acts as a reliable transparent data buffer state line.
    Equation: Q_next = D.
  • T (Toggle) Flip-Flop: Toggles state if input is high, holds state if low.
    Equation: Q_next = T ⊕ Q.

12. Race Around Conditions & Solutions

The Problem Architecture
  • Occurs within a **level-triggered JK Flip-Flop** when J = 1, K = 1, and the clock pulse width (t_w) is longer than the propagation delay of the internal logic gates (t_pd). The output toggles repeatedly and uncontrollably within a single clock cycle, leaving the final state unpredictable.
The Three Standard Solutions
  • 1. Enforce strict **Edge Triggering** (relying on differentiator circuits).
  • 2. Enforce structural **Master-Slave JK Flip-Flop** design configurations.
  • 3. Ensure the clock width is strictly bounded: t_pd < t_w < T_clock.

13. Counter Designs (MOD Boundaries)

MOD Limits
  • An n-bit binary counter can scale to a maximum count sequence of 2ⁿ states (MOD-2ⁿ capacity spanning from state 0 to 2ⁿ-1).
Specialized Shift Counters
  • Ring Counter: A basic shift register loop configuration where the output of the final stage feeds back into the first stage. Uses **One-Hot Encoding** (only one flip-flop is active at a time). An n-bit ring counter provides exactly n distinct states (highly inefficient state density).
  • Johnson Counter (Twisted Ring / Switch-Tail): The inverted output of the final stage feeds back into the first stage input. An n-bit Johnson counter provides exactly 2n distinct states.

14. Registers & Shift Configurations

Core Operations
  • Registers are built by linking multiple edge-triggered flip-flops in series to store multi-bit words.
  • I/O Layout Strategies: SISO (Serial-In Serial-Out), SIPO (Serial-In Parallel-Out), PISO (Parallel-In Serial-Out), PIPO (Parallel-In Parallel-Out).
  • Mathematical Scaling: Shifting bits exactly one position to the left effectively **multiplies the decimal value by 2**. Shifting bits exactly one position to the right effectively **divides the decimal value by 2** (integer division).

16. Non-Volatile ROM Architectures

Classification Matrix
  • PROM (Programmable ROM): Manufactured as an unprogrammed blank matrix that can be written to exactly once via field programmers.
  • EPROM (Erasable PROM): Can be erased and rewritten. Erasing requires exposing the chip to concentrated **Ultra-Violet (UV) light** through a quartz window window.
  • EEPROM (Electrically Erasable PROM): Can be erased and rewritten in-circuit using localized electrical pulses. Features fine byte-level structural modification access.

17. Timing Parameters & Propagation Delays

Core Delay Metrics
  • Propagation Delay (t_pd): The time it takes for a change at the input terminals to cause a steady change at the output pins.
  • Rise Time (t_r): The time required for a signal voltage transition to climb from 10% up to 90% of its maximum peak amplitude value.
  • Fall Time (t_f): The time required for a signal voltage transition to drop from 90% down to 10% of its maximum amplitude value.

18. Logic Hazards & Glitch Triggers

Hazard Classifications
  • Static 1 Hazard: An output that is supposed to remain at a steady 1 momentarily drops to 0 due to asymmetric gate delays during an input change.
  • Static 0 Hazard: An output that is supposed to remain at a steady 0 momentarily spikes to 1.
  • Dynamic Hazard: An output that is supposed to transition cleanly from 0 to 1 (or 1 to 0) fluctuates or bounces multiple times before settling.
Common MCQ Solution Trick
  • Static hazards can be identified on a K-map by looking for adjacent groups that do not overlap. They can be eliminated by adding **redundant look-ahead prime implicant groups** to cover those adjacent boundaries.

19. Integrated Logic Families

Core Feature Identification Metrics
  • TTL (Transistor-Transistor Logic): Built using bipolar junction transistors. Standard classic design baseline. Fan-out typically matches a factor of 10.
  • CMOS (Complementary Metal-Oxide Semiconductor): Employs complementary pairs of PMOS and NMOS transistors. Features near-zero static power consumption and exceptionally high noise immunity.
  • ECL (Emitter-Coupled Logic): Operates based on unsaturated differential amplifier transistor logic states. **The absolute fastest logic family** because it avoids transistor storage delay times, but consumes high power.

21. Tri-State Logic Systems

The Three Legal Functional States
State 0 (Low)  |  State 1 (High)  |  High Impedance (State Z)
  • High Impedance (State Z): The output pin behaves like an open circuit, completely disconnecting it from the signal line. This allows multiple circuit outputs to safely share a common communication line or system bus without causing short circuits.

22. Multiplexers (MUX Data Selectors)

Structural Configuration Scaling Equation
Total Input Data Lines = 2ⁿ  ↔  Requiring exactly 'n' independent Select Control Lines
  • Acts as a digital **Many-to-One** data switch selector.
  • Universal Logic Capability Trap: A 2ⁿ-to-1 MUX can implement any arbitrary boolean logic function of up to n + 1 separate variables without needing any extra logic gates.

23. Demultiplexers (DEMUX Data Distributors)

Operational Logic
  • The precise functional reverse of a MUX system. Operates as a **One-to-Many** data distributor circuit layout. Takes one single input line and routes it to one of 2ⁿ output lines based on the binary state of n select lines.
  • A DEMUX with an active enable pin can be used directly as a binary **Decoder**.

24. Encoders & Priority Resolved Logic

Functional Mapping
  • Decoder: Converts an n-bit coded input into a maximum of 2ⁿ unique output lines.
  • Encoder: Reverses decoder operations. Converts a maximum of 2ⁿ active input lines back into an n-bit coded output word.
  • Priority Encoder ⭐: Addresses the ambiguity trap that occurs if multiple input lines go high simultaneously. It evaluates inputs based on a predefined priority hierarchy and generates the binary code of only the highest-priority active input line.

25. High-Frequency Boolean Simplification Identities

Essential Equation Simplifications
Identity 1: A + A'B = A + B

Identity 2: A(A' + B) = AB

Identity 3 (Distributive Law Extensions): (A + B)(A + C) = A + BC

⚠ Critical CIL MT Digital Logic MCQ Traps Checklist

  • XOR is strictly Associative and Commutative: (A ⊕ B) ⊕ C = A ⊕ (B ⊕ C).
  • NAND and NOR gates are universal; any logic circuit can be built entirely from either gate type.
  • 2's complement subtraction eliminates the need for end-around carry hardware, which is required in 1's complement math.
  • K-map cells must be arranged in **Gray code order** to preserve structural adjacency via single-bit changes.
  • Latches operate on continuous **level triggering**; Flip-Flops depend on precise **edge triggering**.
  • DRAM uses capacitors that leak charge, requiring continuous refresh cycles. SRAM uses stable transistor latches and does not require refreshes.
  • Carry Lookahead Adders perform fast additions with constant delay O(1), bypassing the slow ripple propagation of basic adders.
  • Race around conditions ruin state predictability in level-triggered JK flip-flops when J = 1, K = 1 and the clock pulse is too wide.
  • Tri-state buffers allow safe multi-device communication line sharing by utilizing a high-impedance (State Z) mode to disconnect inactive outputs.
  • A MOD-N counter contains exactly N distinct states (0 to N-1) and resets back to 0 immediately upon hitting its count limit.
