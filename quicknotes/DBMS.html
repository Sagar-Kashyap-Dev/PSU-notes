1. DBMS Basics

Facts / Points
  • DBMS = software for storing and managing data.
  • Reduces redundancy.
  • Ensures consistency.
  • Supports data sharing.
  • Provides security and recovery.
Common MCQ Traps
  • File system → high redundancy.
  • DBMS → controlled redundancy.
  • Data independence is a key feature of DBMS.
Formulas

None.

2. Database Architecture

Facts / Points
  • Three-Level Architecture:
    • External Level (View Level)
    • Conceptual Level (Logical Level)
    • Internal Level (Physical Level)
Common MCQs
  • Physical Data Independence: Change internal schema without affecting conceptual schema.
  • Logical Data Independence: Change conceptual schema without affecting external schema.
Formulas

None.

3. ER Model

Facts / Points
  • Entity: Real-world object.
  • Attribute: Property of entity.
  • Relationship: Association between entities.
  • Types of Attributes: Simple, Composite, Single-valued, Multi-valued, Derived.
Common MCQs
  • Primary Key: Uniquely identifies tuple.
  • Candidate Key: Can become primary key.
  • Super Key: Uniquely identifies tuple (may contain extra attributes).
Formulas

None.

4. Keys

Facts / Points
  • Primary Key: Unique, Not NULL.
  • Foreign Key: References primary key of another table.
  • Candidate Key: Minimal super key.
  • Alternate Key: Candidate key not selected as primary.
Common MCQs
  • Every primary key is a candidate key.
  • Every candidate key is a super key.
  • Every super key is NOT a candidate key.
Formulas

None.

6. SQL

Facts / Points
  • DDL: CREATE, ALTER, DROP, TRUNCATE
  • DML: INSERT, UPDATE, DELETE
  • DQL: SELECT
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK, SAVEPOINT
Common MCQs
  • DELETE: Can be rolled back.
  • TRUNCATE: Faster than DELETE. Removes all rows.
  • DROP: Removes table structure and data.

7. Joins

Facts / Points
  • Inner Join: Matching rows only.
  • Left Join: All rows from left table.
  • Right Join: All rows from right table.
  • Full Outer Join: All rows from both tables.
  • Self Join: Table joined with itself.
Common MCQs
  • Natural Join: Removes duplicate join columns.

8. Functional Dependency

Facts / Points
  • A → B means: A determines B.
  • Types: Full Functional Dependency, Partial Dependency, Transitive Dependency
Common MCQs
  • Used in normalization.
Formulas
Armstrong's Axioms:
- Reflexivity
- Augmentation
- Transitivity

9. Normalization

Facts & Traps
  • 1NF: Atomic values only. Trap: No repeating groups.
  • 2NF: Must be in 1NF. Remove partial dependency. Trap: Applies mainly when composite key exists.
  • 3NF: Must be in 2NF. Remove transitive dependency.
  • BCNF: Stronger than 3NF. Condition: For every FD X → Y, X must be a super key.
Common MCQ Order
1NF → 2NF → 3NF → BCNF

10. Transactions

Facts / Points
  • Transaction: Logical unit of work.
  • ACID Properties:
    • A = Atomicity
    • C = Consistency
    • I = Isolation
    • D = Durability
Common MCQs
  • Commit: Makes changes permanent.
  • Rollback: Undoes transaction.

11. Concurrency Control

Facts / Points
  • Problems: Lost Update, Dirty Read, Unrepeatable Read, Phantom Read
Common MCQs
  • Dirty Read: Reading uncommitted data.
Formulas

None.

12. Serializability

Facts / Points
  • Serial Schedule: One transaction at a time.
  • Concurrent Schedule: Multiple transactions.
  • Conflict Serializable: Equivalent to serial schedule.
Common MCQs
  • Conflict Serializability is checked using a precedence graph.

13. Locking

Facts / Points
  • Shared Lock (S): Read only.
  • Exclusive Lock (X): Read + Write.
Common MCQs
  • Two-Phase Locking (2PL):
    • Growing Phase: Acquire locks.
    • Shrinking Phase: Release locks.

14. Deadlock

Facts / Points
  • Occurs when processes wait indefinitely.
  • Necessary Conditions:
    • Mutual Exclusion
    • Hold and Wait
    • No Preemption
    • Circular Wait
Common MCQs
  • All four conditions must hold simultaneously.

15. Indexing

Facts / Points
  • Primary Index
  • Secondary Index
  • Clustered Index
  • Non-Clustered Index
Common MCQs
  • Index improves: Search speed
  • Index increases: Storage requirement

16. B-Tree

Facts / Points
  • Balanced tree.
  • Used in indexing.
  • All leaves at same level.
Common MCQs (Complexities)
  • Search: O(log n)
  • Insert: O(log n)
  • Delete: O(log n)

17. B+ Tree

Facts / Points
  • Data stored only in leaf nodes.
  • Leaves linked together.
  • Better for range queries.
Common MCQ
  • Difference from B-Tree: B+ Tree internal nodes contain only keys.

18. RAID

Facts / Points
  • RAID 0: Striping | No redundancy
  • RAID 1: Mirroring | High reliability
  • RAID 5: Distributed parity
Common MCQs
  • Highest reliability → RAID 1
  • Best performance → RAID 0

19. File Organization

Facts / Points
  • Heap File: Unordered
  • Sequential File: Sorted
  • Hash File: Hash function used
Common MCQs
  • Fast equality search → Hashing
  • Range queries → B+ Tree / Sequential File

20. DBMS Formulas Worth Memorizing

Formulas
Blocking Factor
bfr = ⌊ B / R ⌋

Where: B = Block Size, R = Record Size

Number of Blocks
b = ⌈ r / bfr ⌉

Where: r = number of records

Indexing Types
  • Dense Index: One index entry per record.
  • Sparse Index: One index entry per block.
