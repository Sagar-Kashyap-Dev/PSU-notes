1. Computer Network Basics

Facts / Points
  • Computer Network: Interconnected devices that communicate and share resources.
  • Core Communication Requirements: Sender, Receiver, Medium, Protocol.
Key Advantages
  • Resource sharing, Seamless Communication, High Reliability, and Scalability.
Common MCQ Traps
  • Internet = A global network of networks.
  • Protocol = Formally defined set of rules governing communication.
Formulas

None.

2. Types of Networks

Facts / Points
  • PAN: Personal Area Network (Smallest coverage footprint, e.g., Bluetooth).
  • LAN: Local Area Network (Office/Home range).
  • MAN: Metropolitan Area Network (Citywide range).
  • WAN: Wide Area Network (Country/Global range, e.g., Internet).
Common MCQs
  • Smallest coverage scale → PAN
  • Largest coverage scale → WAN

3. Network Topologies

Structural Layouts
  • Bus: Single backbone central cable architecture. Single point of failure.
  • Star: Connected directly via a central Hub or Switch.
  • Ring: Data circulates in a strict closed loop.
  • Mesh: Every single node connects directly to every other node (Fully redundant).
Common MCQs
  • Most structurally reliable/fault-tolerant → Mesh Topology
  • Most common in modern commercial deployments → Star Topology

4. OSI Model

Layers & Functions (Top to Bottom)
  • Application: Direct user interfaces and network services.
  • Presentation: Data formatting, encryption, decryption, and compression.
  • Session: Authentication, session setup, maintenance, and teardown.
  • Transport: End-to-end delivery, flow control, error recovery (Segments).
  • Network: Path selection, logical routing, and logical addressing (Packets).
  • Data Link: Reliable node-to-node hop delivery, framing, physical MAC addressing (Frames).
  • Physical: Unstructured bit transmission over physical media (Bits).
Mnemonic

"All People Seem To Need Data Processing"

Common MCQs
  • Routing operations happen at → Network Layer
  • Framing and MAC parsing occur at → Data Link Layer
  • Data encryption/compression happens at → Presentation Layer

5. TCP/IP Model

Core Architecture Layers
  • Application Layer (Combines OSI Layers 5, 6, 7)
  • Transport Layer (Maps directly to OSI Transport)
  • Internet Layer (Maps directly to OSI Network)
  • Network Access Layer (Combines OSI Layers 1, 2)
Common MCQs
  • The TCP/IP Internet Layer is functionally equivalent to the **OSI Network Layer**.
  • Standard reference TCP/IP model has **4 core layers**.

6. Network Devices

Device Mappings
  • Hub: Physical Layer device. Dumb, non-intelligent broadcast mechanism.
  • Switch: Data Link Layer device. Inspects incoming frames and reads MAC addresses.
  • Router: Network Layer device. Selects paths using destination IP addresses.
  • Gateway: Operates across upper layers to perform structural protocol conversions.
Common MCQs
  • Switch decisions depend on → MAC Address
  • Router decisions depend on → IP Address

7. Transmission Media

Guided vs Unguided
  • Guided (Wired): Twisted Pair, Coaxial Cable, Optical Fiber.
  • Unguided (Wireless): Radio Waves, Microwaves, Infrared.
Common MCQs
  • Offers highest structural data bandwidth → Optical Fiber
  • Completely immune to Electromagnetic Interference (EMI) → Optical Fiber

8. Switching Techniques

Core Paradigms
  • Circuit Switching: Establishes a dedicated physical path end-to-end beforehand (e.g., Landline Telephone networks).
  • Packet Switching: Splits data into self-contained packets; paths can vary dynamically.
  • Message Switching: Uses a full complete message framework along with Store-and-Forward hops.
Common MCQs
  • The modern public global Internet relies fundamentally on → Packet Switching

9. Data Link Layer

Functions
  • Framing, Error Detection/Correction, Flow Control, and Physical MAC Addressing.
Common MCQs
  • The Data Link Layer guarantees reliable node-to-node (hop-by-hop) delivery.

10. Error Detection

Methods
  • Parity Bit: Simple single-bit error verification check.
  • Checksum: Accumulator-based verification used heavily within TCP/IP transport headers.
  • CRC (Cyclic Redundancy Check): Polynomial division technique operating at hardware level.
Common MCQs
  • Most reliable mathematical error-detection mechanism → CRC

11. Flow Control

Mechanisms
  • Stop-and-Wait: Sender dispatches exactly one single frame, blocks execution, and waits for ACK before sending next.
  • Sliding Window: Sender can pipeline multiple unacknowledged frames concurrently before blocking.
Common MCQs
  • Sliding Window frameworks yield substantially higher link efficiency than basic Stop-and-Wait.

12. ARQ Protocols

Automatic Repeat Request Types
  • Stop-and-Wait ARQ: Retransmits the single current frame on timeout or NAK.
  • Go-Back-N ARQ: Retransmits the entire window starting from the oldest erroneous unacknowledged frame onward. Receiver discards out-of-order packets.
  • Selective Repeat ARQ: Buffers out-of-order items. Retransmits ONLY the specifically damaged or missing frames.
Common MCQs
  • Most bandwidth-efficient ARQ design pattern → Selective Repeat ARQ

13. MAC Address

Facts / Points
  • 48-bit global address (6 bytes space total).
  • Permanently burned physical address assigned globally by the hardware manufacturer.
Common MCQs
  • MAC addresses are processed at → Data Link Layer (MAC Sublayer)

14. IP Addressing

Versions
  • IPv4: 32 bits long (Expressed as 4 decimal octets separated by dots).
  • IPv6: 128 bits long (Expressed as 8 groups of colon-separated hexadecimal values).
Common MCQs
  • IPv6 addresses are natively formatted and written using **hexadecimal representation**.

15. IPv4 Classful Spaces

Class First Octet Range Default Subnet Mask Primary Purpose
Class A 1 – 126 255.0.0.0 Massive networks (Few networks, huge hosts)
Class B 128 – 191 255.255.0.0 Medium to large enterprise deployments

Class D 224 – 239 N/A Multicasting streams
Class E 240 – 255 N/A Experimental / Research reserves

*Note: 127.x.x.x is intentionally reserved for local loopback testing addresses.

Common MCQs
  • Class D addresses are reserved exclusively for → Multicasting

16. Subnetting

Key Math Layouts
Total Subnets Formed = 2n  (where n = count of borrowed network bits)
Usable Hosts per Subnet = 2h − 2  (where h = remaining host bits)
Common MCQs
  • Why subtract 2 from host count? To account for the unassignable **Network Address** (first address) and the dedicated **Subnet Broadcast Address** (last address).

17. Routing

Facts / Points
  • Routing: Dynamic or structural path selection process across intermediary network hubs.
  • Static Routing: Paths configured manually by network administrators.
  • Dynamic Routing: Protocols calculate optimal paths automatically based on state metrics.
Common MCQs
  • Routing decisions and logical map processing are performed by structural **routers**.

18. Routing Protocols

Core Protocols
  • RIP (Routing Information Protocol): Uses Distance Vector routing logic. Relies strictly on Hop Count as its metric. Maximum allowable hop count is **15** (16 denotes infinity/unreachable).
  • OSPF (Open Shortest Path First): Uses Link-State routing logic. Relies on Dijkstra's algorithm to calculate optimal shortest paths.
  • BGP (Border Gateway Protocol): Path Vector protocol used to handle routing between distinct Autonomous Systems (AS).
Common MCQs
  • Core protocol running the public Internet backbone infrastructure → BGP

19. Transport Layer (TCP vs UDP)

TCP Features
  • Connection-oriented stateful workflow. Reliable data stream guarantee. Implements explicit Flow Control and Error Recovery.
UDP Features
  • Connectionless stateless execution. Faster transmission overhead. Zero reliability or sequence delivery guarantees.
Common MCQs
  • HTTP / HTTPS applications require underlying → TCP connections
  • DNS queries (small payload lookups) rely primarily on → UDP overhead

20. TCP Deep Dive

Three-Way Handshake Connection Steps
Client —[ SYN ]→ Server
Client ←[ SYN-ACK ]— Server
Client —[ ACK ]→ Server
Common MCQs
  • TCP guarantees in-order, lossless, reliable data byte-stream delivery.

21. UDP Details

Facts / Points
  • Lacks connection setup handshakes, active flow control windows, or built-in retransmission recovery.
Common MCQs
  • Real-time services (VoIP streaming, online multiplayer gaming, live broadcast feeds) prefer **UDP** due to lower latency.

23. DNS

Facts / Points
  • Domain Name System: Resolves human-readable alphanumeric strings directly into computer-parseable physical IP addresses.
Common MCQs
  • DNS relies on an organized, globally distributed hierarchical structural database framework.

24. Network Security Basics

Core Concepts
  • Firewall: Packet-filtering security mechanism inspecting inbound/outbound payloads based on defined safety criteria rules.
  • Encryption: Standard scramble transformation algorithm protecting data confidentiality.
  • VPN: Virtual Private Network tunnel creating a masked path through public paths.
Common MCQs
  • Filtering traffic based on incoming or outgoing packet headers is handled by a **Firewall**.

27. CN Formulas Worth Memorizing

Link Performance Delays
Transmission Delay (Tt) = Packet Size / Bandwidth

Propagation Delay (Tp) = Distance / Propagation Speed of Medium

Total Node Link Delay = Tt + Tp + Queuing Delay + Processing Delay
Capacity & Allocation Metrics
Bandwidth-Delay Product (BDP) = Bandwidth × Round Trip Time (RTT)

Subnet Host Capacity Calculation = 2h − 2  (where h = host bits count)
