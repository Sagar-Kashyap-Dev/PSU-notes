1. COA Basics

Facts / Points
  • COA deals explicitly with the operational structure and behavior of computer system hardware components.
  • Organization: "How things are connected/interacted" (Hardware signals, interfaces, peripherals).
  • Architecture: "What the system does" (Instruction Set Architecture (ISA), addressing modes, registers visible to the programmer).
Common MCQ Traps
  • CPU = ALU + Control Unit + Internal Registers.
  • ALU executes raw arithmetic and logical bitwise computations.
  • Control Unit generates timing/control signals to direct hardware operations.
Formulas

None.

2. CPU Components & Registers

Core Internal Architectural Registers
  • Accumulator (AC): Holds execution operands and temporary immediate calculation results natively inside the ALU.
  • Program Counter (PC): Holds the explicit memory address of the next instruction scheduled to be fetched.
  • Instruction Register (IR): Holds the actual binary macrocode of the current instruction being decoded/executed.
  • MAR (Memory Address Register): Holds the specific target physical address currently being accessed for read/write.
  • MDR / MBR (Memory Data / Buffer Register): Acts as a temporary hold buffer for data bytes received from or sent directly to memory storage.
Common MCQs
  • PC → Always holds address of the next instruction.
  • IR → Always holds the current execution code block itself.

3. Instruction Cycle

Sequential Phases
Fetch → Decode → Execute → Store (Write Back)
Common MCQs
  • Fetch phase: Transfers raw instruction blocks directly from Main Memory into the CPU's internal Instruction Register (IR).
  • Execute phase: Activates hardware ALU operations or logic pathways to process the decoded code instructions.

4. Instruction Formats

Structural Formats
  • Zero-Address Format: Relies completely on an internal evaluation Stack architecture (e.g., uses implicit PUSH / POP / ADD triggers).
  • One-Address Format: Utilizes an implicit dedicated Accumulator (AC) register for one of its operands.
  • Two-Address Format: Specifies two distinct address fields (Source and Destination mix, e.g., ADD R1, R2).
  • Three-Address Format: Explicitly separates arguments completely into [Destination, Source 1, Source 2] structures.
Common MCQs
  • Most flexible format with least program line counts → Three-Address Instruction
  • Most compact memory footprint encoding per individual instruction → Zero-Address (Stack-based) Instruction

5. Addressing Modes

Modes & Effective Address (EA) Mechanics
  • Immediate Mode: Operand value is explicitly part of the instruction fields itself. (Zero memory access overhead).
  • Direct Mode: Instruction address field directly holds the exact address target location of the operand. (EA = Address Field).
  • Indirect Mode: Instruction address field holds a pointer pointing to another address where the operand resides. Requires multiple pointers memory hops. (EA = Memory[Address Field]).
  • Register Mode: Operand value resides natively within an internal designated general-purpose CPU register. (No memory access).
  • Register Indirect Mode: Designated register contains the memory address pointer to the actual operand. (EA = [Register]).
  • Indexed / Base Register Mode: Target location is evaluated by adding a fixed reference offset to a dynamic tracking register. (EA = Base/Index Register + Offset value).
Common MCQs
  • Fastest overall execution mode → Immediate Addressing
  • Most programmatic flexibility but suffers from double-memory read cycle latency → Indirect Addressing

6. ALU (Arithmetic Logic Unit)

Core Internal Operations
  • Executes standard Arithmetic computations (Addition, Subtraction, Multiplication).
  • Processes fundamental Logical gates (AND, OR, NOT, XOR).
  • Applies structural Bitwise Shift variations (Arithmetic, Logical, Circular rotations).
Common MCQs Trap
  • The ALU is strictly a computational data path. It does NOT handle, schedule, or route system memory allocations or control parameters.

7. Control Unit (CU) Architecture

Design Paradigms
  • Hardwired Control Unit: Implemented via raw combinational logic gate state circuits, flip-flops, and encoders. Fixed logic. Extremely fast execution speed. High design modification difficulty.
  • Microprogrammed Control Unit: Control signals are generated by parsing internal sequences of primitive control instructions (microcode) stored inside a specialized, dedicated high-speed Control Memory. Slower execution step overhead. High flexibility.
Common MCQs
  • Easily modified, updated, or structurally extended → Microprogrammed CU
  • Provides absolute peak execution speeds → Hardwired CU

8. Memory Hierarchy

Architectural Layout Order (Fastest/Costliest → Slowest/Cheapest)
Internal Registers → Cache Memory (SRAM) → Main Memory (DRAM) → Secondary Storage (SSD/HDD)
Common MCQs
  • Fastest memory structure and smallest logical storage block component within a CPU → Registers
  • Largest cost efficiency and overall capacity metric → Secondary Storage

9. Cache Memory Concepts & Mapping

Core Principles
  • Temporarily stores frequently used instruction blocks to maximize processor speed via the principle of Locality of Reference (Spatial and Temporal).
Mapping Strategies
  • Direct Mapping: Each Main Memory block maps exclusively to exactly one specific predetermined Cache line frame. Highly prone to thrashing conflicts. Simple to implement.
  • Fully Associative Mapping: Main Memory blocks can reside anywhere within any vacant Cache line dynamically. Maximizes efficiency but requires highly complex parallel tag matching hardware.
  • K-Way Set Associative Mapping: Memory is split into sets. A block maps to a designated set but can reside anywhere within that set's internal lines (Combines best of both).
Common MCQs
  • Yields optimal hit performance with minimal mapping conflicts → Associative Mapping
  • Least expensive hardware footprint and simplest architecture layout → Direct Mapping

10. Cache Performance Metrics & Formulas

Direct Mapping Frame Association Target
Target Cache Line Index = (Memory Block Number) MOD (Total Lines in Cache Space)
Hit Dynamics Metrics
Hit Ratio = Total Cache Hits / Total Memory Access Requests

Miss Ratio = 1 − Hit Ratio = Total Cache Misses / Total Memory Access Requests
Average Memory Access Time (AMAT) Formula
AMAT = Hit Time + (Miss Rate × Miss Penalty)

11. Main Memory

Core Subdivisions
  • RAM (Random Access Memory): Volatile working workspace. Requires power to preserve logical states. Uses SRAM for fast cache configurations and DRAM for dense main system modules.
  • ROM (Read Only Memory): Non-volatile structural system bios memory. Permanent layout.
ROM Architecture Configurations
  • PROM: Programmable ROM (Writable exactly once).
  • EPROM: Erasable Programmable ROM (Data erased via concentrated Ultra-Violet exposure).
  • EEPROM: Electrically Erasable Programmable ROM (Erased via internal electrical charge pulses).

12. Pipelining

Core Operational Principle
  • Maximizes overall Instruction **Throughput** by overlapping separate sequential processing execution stages concurrently across incoming commands.
Standard Processing Pipeline Stages
Fetch (IF) → Decode (ID) → Execute (EX) → Memory Access (MEM) → Write Back (WB)
Common MCQs Metrics
  • Under ideal conditions without stall bubbles, the maximum architectural speedup achieved via pipelining is mathematically equal to the total count of internal **Pipeline Stages (k)**.
  • Pipelining improves Throughput (completed operations per unit time); it does NOT reduce the individual structural instruction latency time.

13. Pipeline Hazards

Hazard Core Categories
  • Structural Hazard: Resource conflict loops occurring when two separate active pipeline instructions try to use the same physical hardware component simultaneously (e.g., trying to access a single-ported main memory during both Fetch and MEM stages).
  • Data Hazard: Occurs when an instruction depends on the data outcome result of a preceding instruction still active inside the pipeline stages. Includes Write-After-Read (WAR), Read-After-Write (RAW), and Write-After-Write (WAW) dependencies.
  • Control Hazard (Branch Hazard): Caused by unconditional jumps or conditional branch operations that modify the Program Counter. The pipeline cannot safely predict the next fetch target instructions immediately, introducing stall cycles.
Common MCQs
  • Branch operations or program jump statements systematically trigger **Control Hazards**.

14. Pipelining Performance Formula

Speedup (S) Equation
Speedup (S) = Time taken without a pipeline (Tn) / Time taken using a pipeline (Tp)

S = (n × tn) / [ (k + n − 1) × tp ]

Where:
n = Total count of instructions parsed
k = Count of pipeline stages available
tp = Processing clock cycle period of the pipeline

15. RISC vs CISC Architecture

Architectural Attribute RISC (Reduced Instruction Set Computer) CISC (Complex Instruction Set Computer)
Instruction Complexity Simple, elementary instructions executing in a single clock cycle Complex, variable instructions spanning multiple clock cycles
Instruction Size Fixed instruction encoding size (e.g., 32-bit constant) Variable instruction encoding size
Internal Registers Large count of general-purpose internal registers Limited internal register resources available
Addressing Modes Very few, highly optimized simple modes Large variety of complex addressing modes
Memory Interaction Strictly via LOAD / STORE explicit instructions only Direct memory-to-memory data processing support

16. I/O Organization & DMA

Data Transmission Methods
  • Programmed I/O: The processor loops in a busy-wait state continuously checking device status registers. Highly inefficient. CPU is completely locked up.
  • Interrupt-Driven I/O: Device hardware issues an asynchronous interrupt signal when ready. The CPU can handle other computations in the meantime, avoiding idle waiting.
  • DMA (Direct Memory Access): An independent specialized hardware module handles data block transmission streams directly to/from main memory, bypassing the CPU completely.
Common MCQs
  • Fastest overall physical I/O transfer configuration mechanism → DMA
  • When processing an interrupt, the CPU must always systematically save its current state environment flag parameters and PC counter address to the **Stack**.

17. Computer Arithmetic & Representation

Floating Point IEEE 754 Standard Layouts
  • Used globally to map precision real decimal numbers inside computer hardware environments.
  • Core Split Fields: [ Sign Bit (S) | Biased Exponent (E) | Mantissa / Fraction (M) ]
  • Single Precision (32-bit total): 1 Sign bit + 8 Exponent bits + 23 Mantissa bits.
  • Double Precision (64-bit total): 1 Sign bit + 11 Exponent bits + 52 Mantissa bits.
Sign/Complement Representations
  • 1's Complement: Inverts all raw binary bits (0 → 1, 1 → 0).
  • 2's Complement: Formed by adding 1 directly to the least significant bit (LSB) of the 1's complement representation. (2's Comp = 1's Comp + 1).
Common MCQs
  • Modern computer system hardware uses **2's complement representation** universally to process integer arithmetic subtraction and sign validation tasks.

18. System Performance Metrics & Formulas

Total CPU Execution Time Formula
CPU Time = Instruction Count (IC) × Cycles Per Instruction (CPI) × Clock Cycle Time (τ)

Or:
CPU Time = [ Instruction Count × CPI ] / Clock Rate (f)
CPI & MIPS Calculation Mappings
Average CPI = Total Clock Cycles Spent / Total Executed Instructions Count

MIPS (Millions of Instructions Per Second) = Instruction Count / (Execution Time × 106)

Alternatively:
MIPS = Clock Frequency (f) / (CPI × 106)

⚠ Critical CIL MT COA MCQ Traps Checklist

  • CPU Core = ALU + Control Unit + Internal Register Matrices.
  • PC holds the evaluation address pointer for the next scheduled instruction; IR locks down the raw machine code of the current running instruction step.
  • RISC enforces a fixed instruction layout footprint alongside an isolated Load/Store pipeline design pattern.
  • CISC focuses on microcode-backed multi-cycle variable complex instruction lengths.
  • Hardwired CU operates via fast, fixed combinational hardware loops; Microprogrammed CU runs via highly modifiable, slower control memory storage lookups.
  • Internal Registers constitute the absolute top performance tier in the global storage speed index.
  • Cache optimizations rely explicitly on the Principle of **Locality of Reference** to maximize system speed metrics.
  • Direct mapping handles cache index matching using simple mathematical modulus logic, but is highly vulnerable to mapping block conflicts.
  • AMAT explicitly sums raw hit latency against calculated miss rate penalties: AMAT = Hit Time + (Miss Rate × Miss Penalty).
  • Pipelining scales operation **Throughput** metrics; it does **NOT** improve individual instruction execution latency times.
  • Pipeline execution issues map down to three distinct hazard categories: **Structural** (hardware resource limits), **Data** (read/write dependencies), and **Control** (branching updates to the PC).
  • DMA operations offload block transfers directly onto memory data paths, freeing up the CPU.
  • Interrupt validation routines require the processor to instantly cache working environment flags and current PC values directly into system **Stack** frames.
  • Every real computing environment processes negative operations and binary subtractions via **2's Complement Arithmetic**.
  • IEEE 754 maps single-precision parameters into a distinct 32-bit field pattern: 1 sign bit, 8 biased exponent bits, and 23 fractional mantissa bits.
