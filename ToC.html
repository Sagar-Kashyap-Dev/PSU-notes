Theory of Computation Specification

Formal language transitions, state-space reduction models, context-free normalizations, and structural limits of undecidability.

1. Regular Languages & Finite Automata Engine

DFA: $\delta: Q \times \Sigma \to Q$ Mapping
NFA: $\delta: Q \times \Sigma \to 2^Q$ Mapping
$\epsilon$-NFA: $\delta: Q \times (\Sigma \cup \{\epsilon\}) \to 2^Q$
Equivalence: $\mathcal{L}(\text{DFA}) = \mathcal{L}(\text{NFA})$
Formal Mathematical Definitions

A Finite Automaton is formally defined as a 5-tuple $M = (Q, \Sigma, \delta, q_0, F)$ where:

  • $Q$ is a finite, non-empty set of operational machine states.
  • $\Sigma$ is a finite alphabet of terminal input symbols.
  • $q_0 \in Q$ is the designated initial state.
  • $F \subseteq Q$ is the set of final/accepting states.
  • Transition Function ($\delta$) Architectures:
    • *Deterministic Finite Automaton (DFA):* $\delta: Q \times \Sigma \to Q$ (Exactly one path exists for each input).
    • *Nondeterministic Finite Automaton (NFA):* $\delta: Q \times \Sigma \to 2^Q$ (Zero, one, or multiple paths can be evaluated simultaneously).
    • *NFA with $\epsilon$-transitions ($\epsilon$-NFA):* $\delta: Q \times (\Sigma \cup \{\epsilon\}) \to 2^Q$ (Allows spontaneous transitions without consuming an input symbol).

2. DFA Minimization Framework

Myhill-Nerode: Equivalence Class Slicing
0-Equivalence: Separate $F$ from $(Q \setminus F)$
k-Equivalence: Check Transitions into $(k-1)$ sets
Uniqueness: Canonical Minimal DFA
Algorithmic Partitioning Engine (Myhill-Nerode / Hopcroft)

Minimizing a DFA merges equivalent states to construct a unique, minimal state-space framework. States $p, q$ are equivalent if for all strings $w \in \Sigma^*$, $\hat{\delta}(p, w) \in F \iff \hat{\delta}(q, w) \in F$.

  1. Initial Partition ($\Pi_0$): Divide states into final and non-final sets: $\Pi_0 = \{F, Q \setminus F\}$.
  2. Iterative Step ($\Pi_{k+1}$): Split an existing block $B$ if a pair of states $p, q \in B$ transition to distinct blocks under any input symbol $a \in \Sigma$: $$\text{If } \delta(p, a) \in B_i \text{ and } \delta(q, a) \in B_j \quad (B_i \neq B_j), \text{ split } B.$$
  3. Terminate when $\Pi_{k+1} = \Pi_k$. Each remaining partition block forms a single consolidated state in the minimal DFA.

3. Regular Expressions & The Pumping Lemma Rule

Decomposition: $w = xyz$ String Splits
Length Limit: $|xy| \le p$ Precondition
Non-Empty: $|y| \ge 1$ Loop Constraint
Pumping Condition: $\forall i \ge 0, xy^iz \in L$
The Pumping Lemma for Regular Languages Proof Theorem

Used to prove a language $L$ is **not regular** (cannot be used to prove a language is regular). If $L$ is regular, there exists a pumping length $p$ such that any string $w \in L$ with $|w| \ge p$ can be split into three parts, $w = xyz$, satisfying:

  • $|y| \ge 1$ (The loop segment cannot be empty).
  • $|xy| \le p$ (The first two components must fall within the pumping length).
  • $$xy^iz \in L \quad \text{for all } i \ge 0$$

Standard Application Example: Denying regularity for $L = \{a^n b^n \mid n \ge 0\}$ by picking $w = a^p b^p$. Since $|xy| \le p$, $y$ consists entirely of $a$ symbols. Pumping $y$ changes the count of $a$s without changing the count of $b$s, driving the resulting string outside the language definition.

4. Context-Free Grammars & Structural Parse Trees

4-Tuple CFG: $G = (V, \Sigma, P, S)$ Definition
Derivation: Leftmost vs. Rightmost Sequences
Ambiguity: $\ge 2$ Distinct Parse Trees
Inherent Ambiguity: No Unambiguous Grammar Exists
Grammar Definitions & Parse Trees

A Context-Free Grammar is defined as $G = (V, \Sigma, P, S)$ where production rules follow the constraint $A \to \alpha$ ($A \in V$ and $\alpha \in (V \cup \Sigma)^*$).

  • Ambiguity: A grammar $G$ is ambiguous if there exists a string $w \in \mathcal{L}(G)$ that yields **two or more distinct leftmost derivations (LMDs)**, rightmost derivations (RMDs), or parse trees.
  • Inherent Ambiguity: A context-free language $L$ is inherently ambiguous if *every possible context-free grammar* that generates $L$ is structurally ambiguous (e.g., $L = \{a^i b^j c^k \mid i=j \text{ or } j=k\}$).

5. Simplification & Context-Free Normal Forms

Useless Symbols: Non-Generating or Non-Reachable
$\epsilon$-Productions: $A \to \epsilon$ Null Transformations
Unit Productions: $A \to B$ Single Variables
CNF Constraints: $A \to BC$ or $A \to a$ Rules
GNF Constraints: $A \to a\alpha$ ($\alpha \in V^*$) Rules
Simplification Strategy Pipeline

To clean a grammar before normalization, remove non-generating and unreachable variables, followed by resolving null ($\epsilon$) and unit productions:

$$\text{Grammar } G \longrightarrow \text{Remove Nullable Steps} \longrightarrow \text{Eliminate Unit Chain Steps} \longrightarrow \text{Prune Useless Subtrees}$$
Formal Normal Forms Reference Structural Definition
  • Chomsky Normal Form (CNF): All production rules are strictly constrained to the formats: $$A \to BC \quad \text{or} \quad A \to a$$ *Where $A, B, C \in V$ and $a \in \Sigma$. Deriving a string of length $n$ using a CNF grammar requires exactly **$2n - 1$ derivation steps**.*
  • Greibach Normal Form (GNF): All production rules are strictly constrained to the format: $$A \to a\alpha$$ *Where $a \in \Sigma$ and $\alpha \in V^*$ (Starts with exactly one terminal symbol, followed by zero or more variables).*

6. Pushdown Automata (PDA & Stack Memory)

7-Tuple PDA: $(Q, \Sigma, \Gamma, \delta, q_0, Z_0, F)$
DPDA: $\le 1$ Deterministic Path Mapping
NPDA: Power-Engine Context Free System
Acceptance: Final State vs. Empty Stack
Transition Mechanics & Stack Interfacing

PDAs add an infinite LIFO stack to finite automata. Formally defined with transition map $\delta: Q \times (\Sigma \cup \{\epsilon\}) \times \Gamma \to 2^{Q \times \Gamma^*}$.

  • Deterministic PDA (DPDA) vs. Nondeterministic PDA (NPDA): Unlike finite automata, NPDAs are strictly more powerful than DPDAs. NPDAs accept all Context-Free Languages (CFLs), whereas DPDAs accept only **Deterministic CFLs (DCFLs)**. This means languages like the set of even-length palindromes ($L = \{ww^R \mid w \in \{a,b\}^*\}$) require nondeterminism to locate the center axis point.

7. Turing Machines & Computability Foundations

7-Tuple TM: $(Q, \Sigma, \Gamma, \delta, q_0, B, F)$
Transition Map: $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$
Halting: Acceptance by Entering Stable Stop
Church-Turing: Effective Computability Definition
The Universal Machine Engine Architecture

A Turing Machine features an infinite read/write tape divided into cells containing symbols from tape alphabet $\Gamma$. Its execution path is driven by transition map $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$.

[Image diagram of a formal Turing Machine model showing the finite control state unit, the read-write head, and the infinite tape with blank symbols]
  • Deterministic vs. Nondeterministic Turing Machines: Nondeterministic Turing Machines (NTMs) **do not possess extra computational power** over Deterministic Turing Machines (DTMs). Every NTM can be simulated by an equivalent DTM, though it may require an exponential step-count increase.
