1. Number Representation & Signed Arithmetic

IEEE 754 Single: 1 Sign | 8 Exp | 23 Mantissa
IEEE 754 Double: 1 Sign | 11 Exp | 52 Mantissa
Bias Constant: $2^{k-1} - 1$ Factor
Booth Recoding: Bit-Pair Multiplier Reduction
IEEE 754 Floating-Point Framework

Normal floating-point values are structurally calculated as follows:

$$\text{Value} = (-1)^{\text{Sign}} \times (1.\text{Mantissa}) \times 2^{\text{Exponent} - \text{Bias}}$$
  • Single-Precision (32-bit): Bias = 127. Exponent range `00000001` ($1$) to `11111110` ($254$). `00000000` is reserved for subnormal numbers/zero; `11111111` is reserved for $\pm \infty$ and NaN.
  • Double-Precision (64-bit): Bias = 1023.
Booth Multiplier Algorithm Logic

Enables efficient multiplication of signed 2's complement integers by looking at adjacent bit pairs $Q_n$ and $Q_{n+1}$ (initially $Q_{n+1} = 0$):

Current Bit ($Q_n$)Previous Bit ($Q_{n+1}$)ALU Operational Response Action
00No algebraic operations. Shift right arithmetically.
01Add Multiplicand ($M$) to Accumulator ($A$). Shift right arithmetically.
10Subtract Multiplicand ($M$) from Accumulator ($A$). Shift right arithmetically.
11No algebraic operations. Shift right arithmetically.

2. CPU Internal Organization & Instruction Cycles

Program Counter: Next Address Reference
Instruction Reg: Opcode Storage Buffer
MAR / MDR: Bus Hardware Ports
Control Unit: Hardwired vs. Microprogrammed
The Hardware Register Assembly
  • PC (Program Counter): Tracks the main memory address of the next instruction slated to execute.
  • IR (Instruction Register): Holds the binary pattern of the instruction currently being decoded.
  • MAR (Memory Address Register) & MDR (Memory Data Register): Interface directly with the physical system buses to coordinate reading or writing memory.
Sub-Cycle Execution Breakdown

Every instruction travels through a distinct state machine phase sequence:

$$\text{Fetch} \longrightarrow \text{Decode} \longrightarrow \text{Evaluate Effective Address} \longrightarrow \text{Execute Data Operations} \longrightarrow \text{Write-Back}$$
Control Unit Architectures Comparison
  • Hardwired Control Unit: Built using combinational and sequential hardware circuits (logic gates, decoders). It is extremely fast but inflexible, meaning any change to the instruction set requires redesigning the physical layout.
  • Microprogrammed Control Unit: Uses low-level control code stored in a dedicated **Control Memory (ROM)**. It updates execution behavior via code modification rather than hardware alterations, though it incurs a latency penalty for memory lookups.

4. Cache Memory Optimization Core

Direct Map: Line = Block Modulo Cache-Lines
Fully Associative: Place Block in Any Line Slot
Set Associative: Set = Block Modulo Total-Sets
Miss Penalty: Main Memory Fetch Overhead Delay
Address Field Bit-Slicing Architectures

Main memory blocks are mapped into cache lines by slicing the physical address bits into distinct hardware fields:

[Image diagram of physical address field bit-splitting layout comparing Direct Mapping tag-line-word fields with Set-Associative tag-set-word fields]
  • Direct Mapping Fields: $\langle \text{Tag}, \text{Line Index}, \text{Block Offset} \rangle$
    Line Index size is bounded by the total number of lines: $$\text{Lines} = \frac{\text{Cache Size}}{\text{Block Size}}$$
  • $k$-Way Set Associative Mapping Fields: $\langle \text{Tag}, \text{Set Index}, \text{Block Offset} \rangle$
    Set Index size is bounded by the total number of sets: $$\text{Sets} = \frac{\text{Cache Size}}{k \times \text{Block Size}}$$
  • Fully Associative Mapping Fields: $\langle \text{Tag}, \text{Block Offset} \rangle$ (Blocks can reside in any cache line slot; tags are searched in parallel using content-addressable hardware).
Performance Equation Matrices

Let $h$ represent cache hit ratio, $T_c$ denote internal cache lookup access latency, and $T_m$ denote main memory access latency:

  • Simultaneous Access Average Access Latency: $$T_{\text{avg}} = h \cdot T_c + (1 - h) \cdot T_m$$
  • Hierarchical / Hierarchical Access Average Access Latency: $$T_{\text{avg}} = T_c + (1 - h) \cdot T_m$$

5. Memory Hierarchy & Interleaving

SRAM: 6-Transistor Internal Cache Cell
DRAM: 1-Capacitor Refresh-Driven Memory Cell
Interleaving: Parallel Module Memory Structuring
Disk Latency: Seek + Rotational + Transfer
Main Memory Technologies
  • SRAM (Static RAM): Built using multi-transistor cross-coupled latches (typically 6 transistors per cell). Fast and does not require periodic refresh cycles. Used for CPU caches.
  • DRAM (Dynamic RAM): Stores bits as charge on a single capacitor paired with an access transistor. It is highly dense and inexpensive, but requires periodic refresh cycles to counteract charge leakage. Used for system main memory.
Memory Interleaving Throughput Engine

Distributes consecutive memory addresses across distinct physical memory modules to support overlapping access cycles:

  • Low-Order Interleaving: Uses the least significant bits of the address to select the memory module. This maps consecutive memory addresses to consecutive modules, allowing parallel pipelined access to consecutive data blocks.

6. I/O Interfaces, Interrupts & DMA Cores

Programmed I/O: Polling-Driven CPU Stalls
Interrupt I/O: Asynchronous Event Signals
DMA Controller: Bus Master Cycle Stealer
Vectored Interrupt: Hardware Table Address Vector
Data Transfer Operational Control Mechanics
  • Programmed I/O: The CPU repeatedly checks device status registers in a loop ("polling"). This wastes CPU clock cycles in idle states.
  • Interrupt-Driven I/O: Hardware devices signal the CPU asynchronously over dedicated lines when data is ready. The CPU suspends its current execution path, saves its state context, and jumps to the corresponding **Interrupt Service Routine (ISR)**.
  • Direct Memory Access (DMA): Relieves the CPU of high-volume data transfer overhead. The DMA controller takes control of the system buses from the CPU ("cycle stealing" or "burst block mode") to transfer data directly between I/O peripherals and main memory.

7. Instruction Pipelining & Performance Hazard Profiles

Structural Hazard: Hardware Resource Collision
Data Hazard: RAW | WAR | WAW Dependencies
Control Hazard: Branch Misprediction Penalties
Ideal Speedup: $S_n = k$ (Stage Factor Lower Bound)
Pipeline Throughput Mathematics

For an instruction stream executing on a $k$-stage pipeline processing $n$ instructions:

  • Total Execution Time (Ideal Conditions): $$T_k = [k + (n - 1)] \cdot \tau$$ *Where $\tau$ is the pipeline clock period, dictated by the slowest stage delay plus latch overhead:* $\tau = \max(t_{\text{stage}}) + t_{\text{latch}}$.
  • Actual Realized Performance Speedup Metrics: $$S = \frac{\text{Non-Pipelined Execution Time}}{\text{Pipelined Execution Time}} = \frac{n \cdot t_n}{[k + (n - 1)] \cdot \tau + \text{Stall Cycles}}$$ As $n \to \infty$ under zero-hazard conditions, the speedup approaches the number of stages ($S \to k$).
Data Dependency Hazards Taxonomy
  • Read-After-Write (RAW / True Dependency): Instruction $I_j$ tries to read a source register before instruction $I_i$ writes to it. Resolved by **Operand Forwarding** or injecting stall cycles (bubbles).
  • Write-After-Read (WAR / Anti-Dependency): Instruction $I_j$ tries to write to a target register before instruction $I_i$ reads it.
  • Write-After-Write (WAW / Output Dependency): Instruction $I_j$ tries to write to a register before instruction $I_i$ writes to it.

Flynn's Parallel Computing Taxonomy

Classifies processing engines based on the number of concurrent Instruction Streams ($I$) and Data Streams ($D$):

  1. SISD (Single Instruction, Single Data): Standard scalar uniprocessors executing sequential operations.
  2. SIMD (Single Instruction, Multiple Data): Vector or array processors where a single control unit broadcasts the same instruction to multiple processing elements working on distinct data blocks.
  3. MISD (Multiple Instruction, Single Data): Rare structural layout used for fault-tolerant computing systems.
  4. MIMD (Multiple Instruction, Multiple Data): Distributed parallel computers and modern multi-core chips where independent processors concurrently execute distinct instruction streams on distinct data pools.
