Dequeue: $\mathcal{O}(1)$		 Tracks discovered nodes sequentially to process the graph level-by-level (FIFO ordering). Using a **Stack** switches the traversal mechanism from level-by-level BFS to depth-first DFS.
DFS (Depth-First Search) LIFO Stack Push: $\mathcal{O}(1)$
Pop: $\mathcal{O}(1)$		 Tracks the current path across deep processing lines (reused via the operating system run stack or explicit structures). Fails on Space: Can throw a stack overflow on deep, linear graph topologies if memory limits are reached.

Decrease-Key: $\mathcal{O}(\log V)$		 Extracts the unvisited vertex with the minimum tentative distance estimate. An unindexed **Array** scales poorly to $\mathcal{O}(V^2)$ on extraction phases. Fibonacci heaps optimize decrease-key operations to amortized $\mathcal{O}(1)$.
Kruskal’s MST Disjoint-Set (Union-Find) Find: $\mathcal{O}(\alpha(V))$
Union: $\mathcal{O}(\alpha(V))$		 Tracks connectivity partitions to check if adding an edge will introduce a cycle. Using standard DFS cycle checks forces processing limits down to an inefficient $\mathcal{O}(E \cdot V)$ runtime.

3. Pattern Matching & Associative Keyword Systems

String & Search Tree Mappings

Rabin-Karp Pattern Match Hash Map / Rolling Hash Hash Re-computation: $\mathcal{O}(1)$ Computes numerical hash signatures over text windows to filter out mismatching substrings quickly. Degrades to $\mathcal{O}(N \cdot M)$ if hash collisions occur frequently due to a poorly chosen prime modulus.
Prefix Auto-Complete Trie (Prefix Tree) Character Lookup: $\mathcal{O}(L)$ (length $L$) Stores string paths along shared character nodes to enable fast prefix lookups. Using a **Hash Map** allows exact matches but cannot perform prefix scanning or range filters efficiently.

4. Fundamental System Operations & Language Runtimes

Compiler Parenthesis Scope Matching LIFO Stack Push / Pop: $\mathcal{O}(1)$ Saves open braces and pairs them against closing braces in reverse order to validate nested structural boundaries. Queues cannot match nested boundaries because they process tokens in a first-in, first-out (FIFO) order.
OS Process Scheduling Queue / Red-Black Tree Min Fetch: $\mathcal{O}(\log N)$ or $\mathcal{O}(1)$ Manages ready processes. Round-Robin scheduling uses FIFO queues; Linux Completely Fair Scheduling (CFS) relies on Red-Black Trees. Linear collections force task lookups to scan the entire set, scaling poorly at $\mathcal{O}(N)$.
Undo-Redo Operations Dual LIFO Stacks Push / Pop: $\mathcal{O}(1)$ One stack records your active history; items are popped and moved to a second stack when rolling back changes. Single linear lists require moving index elements forward and backward, which complicates state tracking.
